# v0.29.6 — Refactor track, increment 6: a `CodeWriter` / `wl!` indentation helper

- **Status:** Draft (proposal). **Merging is approval to build.** *(Refactor track item 9. Independent
  of the `project.rs` chain (`29.2`–`29.5`); its only prerequisite is the `29.1` safety net + the
  existing golden fixtures. Ideally lands **before** `29.7` (`lower_expr`'s statement-emitting
  neighbours) and is incidental to `29.8`. Behaviour-preserving — emitted TS is byte-identical.)*

## Context — infallible-`unwrap` noise and hand-threaded indentation

- The emitter writes TypeScript into a `String` via **123 `writeln!(out, …)` sites in `emitter.rs`
  alone** (506 `write!`/`writeln!` across the whole emitter module:
  `emitter.rs` 172, `serialisation.rs` 169, `workers_entry.rs` 101, `workers.rs` 36, `wrangler.rs`
  28). Every one ends `.unwrap()` — **infallible** noise, since writing to a `String` never fails.
- Indentation is threaded **by hand**: `const INDENT_STEP: usize = 2` (`emitter.rs:35`) is passed as a
  bare `usize` through every emitter (`emit_block_as_function_body(out, …, INDENT_STEP * 2, …)`,
  `emit_doc_block(out, …, indent)`), each function re-deriving `" ".repeat(indent)` (e.g.
  `emit_doc_block`, `:1837`). The depth arithmetic (`INDENT_STEP * 2`) lives at call sites, so the
  indentation contract is diffuse and easy to get subtly wrong.

## Builds on (verified)

- **The pattern is uniform** — `fn …(out: &mut String, …, indent: usize)` + `writeln!(out,
  "{indent_str}…").unwrap()` — which makes the conversion mechanical: a `CodeWriter` owns the `String`
  and the current depth, so `indent_str` and `.unwrap()` both vanish from call sites.
- **`INDENT_STEP` is the single width knob** (`:35`), already centralised — the writer adopts it as
  spaces-per-level, so output width is unchanged by construction.
- **The golden fixtures cover emitted output comprehensively** — the entire point of the emitter is its
  TS output, which the fixtures pin. That is the acceptance gate for a faithful conversion.

## Settled

- **▸ Byte-identical output.** Same spaces, same newlines. The fixtures must pass unedited; any diff is
  a defect, not an update.
- **▸ Retire the infallible `.unwrap()`s and centralise indentation.** The writer makes writing
  total (no `Result`) and owns the depth, so call sites stop doing `" ".repeat(indent)` and
  `INDENT_STEP * n` arithmetic.

## Decisions (open — settle at review)

**[DECISION] Writer shape.** *Rec:* a `CodeWriter` struct owning `String` + `depth: usize`, with
`line(args)` (writes at current depth + newline), `blank()`, and an RAII `indented()` scope guard that
bumps depth and restores on drop — plus a thin `wl!(w, "…")` macro forwarding `format_args!` to
`line` for the format-string ergonomics the current `writeln!` calls rely on. Expose `into_string()` /
`as_str()`. *Alternative:* methods only, no macro — but the macro keeps the 500 call sites a near
mechanical `writeln!(out, X).unwrap()` → `wl!(w, X)` swap, which is the cheapest faithful translation.

**[DECISION] Conversion scope per PR.** 506 sites is a large diff. *Rec:* **convert module-by-module,
one PR each** — `emitter.rs` core first (it neighbours `29.7`), then `serialisation.rs`,
`workers_entry.rs`, `workers.rs`, `wrangler.rs`. Each PR is independently golden-verified; the
`CodeWriter` type lands in the first. *(A single mega-PR is faithful but unreviewable; the per-module
split mirrors `29.2`'s reasoning.)*

**[DECISION] Depth model: absolute spaces vs levels.** Today call sites pass *spaces* (`INDENT_STEP *
2`). *Rec:* the writer tracks **levels** and multiplies by `INDENT_STEP` internally; call sites express
*nesting* (`indented()` scopes) not arithmetic. Where a function is entered at a non-zero base depth
(e.g. `emit_block_as_function_body(.., INDENT_STEP * 2, ..)`), seed the writer's depth from the level,
preserving the exact column. Verify the seam functions reproduce their current base indent.

## Spec delta

**None.** No language surface, no `docs/src/spec/` chapter — an emitter-internal plumbing change.

## Tests

- **No new golden fixtures.** The existing emitter fixtures are the gate; all stay green and unedited.
- **A couple of `CodeWriter` unit tests** for its own contract: nesting via `indented()` produces the
  expected columns, `blank()` emits a bare newline, `INDENT_STEP` width is honoured — cheap, and they
  document the writer for `29.7`/`29.8`.

## Diagnostics / Risks

- **Off-by-one indentation.** The sole real risk is a converted site landing at the wrong depth.
  Mitigation: per-module PRs + golden fixtures catch any column drift immediately; convert the
  base-depth-seeded call sites (`emit_block_as_function_body`, `:2159`/`:2203`/`:2394`/`:2455`, and
  ~8 more) deliberately, checking their fixtures first.
- **Macro hygiene.** `wl!` must forward `format_args!` exactly as `writeln!` did (capture syntax,
  `{x}` interpolation). Pin it with a unit test over a representative format string before the bulk
  swap.
- **Borrow ergonomics.** Methods take `&mut self`; sites that interleave reads of the in-progress
  buffer (rare) need care — but the dominant shape is append-only, so the swap is clean.

## Definition of done

A `CodeWriter` (owning the buffer + indent depth, with `line`/`blank`/`indented` and a `wl!` macro)
replaces the hand-threaded `out: &mut String` + `usize` indent + `.unwrap()` pattern across the emitter
module (landed module-by-module, each PR golden-verified); `INDENT_STEP` becomes the writer's
spaces-per-level so output is byte-identical; a few writer unit tests pin its contract; all emitter
golden fixtures pass unedited; `cargo build`/`test`/clippy and CI stay green; changelog notes the
writer. The implementing PR(s) **delete this file**.
