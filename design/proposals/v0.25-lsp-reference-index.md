# v0.25 — LSP reference index + references & rename (A-0, slice 2)

- **Status:** Draft (proposal). **Merging is approval to build.** *(Second A-tier LSP
  increment; builds the project-wide binding index that `references`/`rename` — and later
  workspace-symbols/document-highlights — consume. Touches `karnc` (resolution sites) +
  `karn-lsp`.)*
- **Builds on (verified):**
  - The LSP advertises `definition` + `hover` (`karn-lsp/src/main.rs:285-286`) — plus
    `completion`, formatting, and `documentSymbol` (`main.rs:290-300`) — but **no
    `references`/`rename`**. `definition` and `hover` are **name-string matching** today
    (`symbols.rs:20-40`, cross-file by parsing each file `:190`) — so both **mis-navigate on
    duplicate names/shadowing**.
  - **Resolution is distributed across three components** — the fact that reshapes the index's
    plumbing:
    - **`resolver.rs`** — type refs, free-`fn` calls, expression identifiers, static/constructor
      calls (`T.method`/`T.of`/variants), qualified variants, record-construction field *names*.
    - **`checker.rs`** — **instance-method receivers** (the resolver recurses into the receiver
      as a value and leaves UFCS dispatch to the checker over `expr_types`; `resolver.rs:1496`,
      `checker.rs:10,146`), plain field reads, and **given-used** tracking.
    - **`project.rs`** — **`given` prefixes** (`resolve_consume_prefix`, ~`:2180`), **flattened
      bare caps** (`:1024`), **provider→capability** links (`:1389`), cross-context wiring.
  - `ResolvedCommons` holds **name → declaration** tables (`resolver.rs:34-52`) but **no
    per-occurrence use→def map** — resolution validates, then discards the binding edges.
  - **v0.24's `ErrorSink`** (`project.rs:308`) — the collection-point attribution mechanism —
    is the precedent the reference-table sink mirrors; the project pass is `diagnose_project`
    (`lib.rs:270`), whose **analysed-snapshot** position rule carries over.
  - The current symbol code **returns no items for `Test`/`Integration` units**
    (`symbols.rs:28`) — a skip the index must **not** inherit (test units name project fns/types).
- **Spec sections touched:** `design/karn-lsp-spec.md` (References + Rename + the capabilities;
  §4.3); `karnc` (the reference-table sink across the resolution sites; the project-pass
  assembly).
- **Scope:** A-0 **slice 2** — a project-wide **binding index** (binding-correct), and its first
  consumers `textDocument/references` and `rename`/`prepareRename`. Document-highlights and
  workspace-symbols fall out of the index but are **out of scope** here.

## Settled (recorded as ADRs ≥0053)

- **▸ The index is binding-correct, built from resolution — never name-matching.** References
  and rename must honour scope/shadowing/aliases; name-matching is already wrong (today's
  `definition`/`hover` prove it). As a **rider**, both **`definition` and `hover`** re-point at
  the index — fixing their shared name-collision bug.
- **▸ Positions convert against the analysed snapshot** (v0.24); rename emits **versioned**
  edits (below).

## Decisions (open — settle at review)

**[DECISION] Reference edges via a sink threaded through *all three* resolution sites.** "The
resolver emits the table" is **wrong**: resolution is distributed across `resolver.rs`,
`checker.rs`, and `project.rs` (above). In particular, three **in-scope** reference kinds resolve
in the **checker**: cross-context capability op-calls `B.Cap.op` / `Alias.Cap.op`
(`resolve_cross_capability`, `checker.rs:4956-4962`), **bare/flattened** capability op-calls
`Cap.op` (`checker.rs:5001-5006`), and cross-context service calls `B.service(args)`
(`checker.rs:4963`). *Rec:* a **reference-table sink** (the analogue of v0.24's `ErrorSink`,
`project.rs:308`) **threaded through all three sites — `resolver.rs` + `checker.rs` +
`project.rs` — recording edges only for in-scope kinds**. What stays **deferred is the symbol
*kinds*** (instance methods, record fields), **not the checker component** — the checker is in the
sink this increment for capability/service op-calls. Correct-by-construction, honest about the
plumbing. The AST/text re-derivation alternative is rejected as before.

**[DECISION] Symbol identity — structural key, qualified at assembly.** *Rec:*
`(unit-qualified-name, kind, name)`. *Wrinkle:* the resolver works on **bare-name merged tables**
(`uses` imports merged in; decls don't carry their defining unit) — only the project driver knows
which unit contributed each name (`unit_uses`, `project.rs:852-904`), and bare-name uniqueness per
importing file is guaranteed by `karn.uses.name_conflict`. So **unit qualification happens at
project-pass assembly**, not at edge-record time. *(Methods, when added, need the owner type in
the key — `(unit, method, Type.name)`.)*

**[DECISION] Reference completeness — aligned to scope.** Capture, for the in-scope kinds:
**type references in both positions** — annotation (`TypeRef::Named`) **and static-receiver /
expression position: `T.of(...)` / `T.method(...)` (`resolver.rs:1465`), `T.Variant`
(`resolver.rs:1352`), constructor `T { … }` (`resolver.rs:1175`)** (the easy-to-miss kind the
matrix exists to catch); `fn` calls + first-class `fn` names; `capability`/`provides`
references; **`given` clauses** (bare `Cap`, dotted `B.Cap`, `consumes U { Cap }` **flattened
bare** caps); capability op-calls — where the recorded edge targets the **capability segment**
(`Cap` in `Cap.op` / `B.Cap.op`; **op names are deferred** with methods, since an op rename also
needs provider-implementation edges); cross-context/aliased calls; service/agent references;
**clause-list occurrences** (all `project.rs`-resolved) — `exports opaque`/`transparent { T }`
and `exports capability { Cap }` entries (`ExportsDecl.names`, `ast.rs:121-123`) and the
`consumes U { Cap, … }` **selection list** (`ConsumesDecl.selected`, `ast.rs:113`), the
declaration-adjacent sites that a rename must edit or it breaks visibility/flattening wiring;
and **references from `test`/`integration` units** (they name project fns/types — skipping them
silently breaks `karn test`). Record the **name-segment span only** (dotted `B.Cap` → `Cap`).
**Dropped from must-capture this increment:** instance-method receivers, plain record/field
reads, and **op names** — all tied to the deferred methods/fields/ops scope.

**[DECISION] Scope of symbols.** *Rec:* top-level named declarations resolved in
`resolver.rs`/`project.rs` — **types, free `fn`s, capabilities, services, agents, providers** —
and their cross-unit references **including from `test`/`integration` units**. **Deferred (with
local bindings):** **methods** (checker-resolved; need the owner-qualified key + the checker sink),
**record fields**, and **op names** (an op rename also needs provider-implementation edges).
**Out of scope:** **unit/context names** — renaming one implies a *file move*
(source-path-mirrors-qualified-name), the A-3 file-operations increment.

**[DECISION] Build on the v0.24 pass.** *Rec:* extend the `diagnose_project` pass to **retain
per-file resolution** and assemble the index **at project-pass assembly** (where unit
qualification is known) — not a second analysis pass.

**[DECISION] `references` + `rename` slice, and how rename is validated.** *Rec:* ship the **index
+ `references` first** to prove the index, then **`rename`** on the proven index (ship both only
if review judges the index complete). **Validate rename two ways, both correct-by-construction:**
**(1) collisions** — apply the candidate `WorkspaceEdit` to an **in-memory overlay** and re-run
**`diagnose_project`**; **refuse if new diagnostics appear** (catches same-unit clash, `uses`
name-conflict `project.rs:1167`, flattened-cap clash `project.rs:1051`, … by construction).
**(2) capture/escape** — re-analysis does *not* catch silent re-binding: renaming `fn a → b` at a
site where a local `b` is in scope **re-binds with no new diagnostic** (bare idents resolve
params-first and locals are callable since v0.20a, `resolver.rs:881-904`). So additionally
**rebuild the index on the edited overlay and assert it is unchanged *modulo the rename*** — the
renamed symbol's old key replaced by the new, **every other symbol's reference set identical**.
This is strictly stronger than (and simpler than) a per-symbol assert: it also catches a re-bind
that flips a *third* symbol's resolution (a shadowed name). Refuse otherwise. Emit **versioned
`TextDocumentEdit`s** so a client rejects a **stale** rename (buffer drifted since the analysed
snapshot) rather than mis-applying it.
**`prepareRename` refuses out-of-scope symbols** — a request on a local binding, a method, an op
name, or a unit/context name returns **null** (no fall-through to a partial or name-matched
rename), so the deferred kinds fail cleanly rather than half-renaming.

## Tests (the LSP analogue of fixtures)

- **`karnc`** *(`cargo test`, no LSP):* a multi-file project exercising the must-capture matrix —
  a cross-file **type** (in **annotation *and* static-receiver position**: `T.of`, `T.Variant`),
  a **`fn`**, a **flattened bare cap**, a **dotted `B.Cap`** op-call, an **`exports`/`consumes`
  clause-list occurrence**, and **a reference from a `test` unit** — assert each maps to its
  **one** definition (two same-named symbols in different units are **not** conflated), with
  **name-segment** spans (dotted `B.Cap` → `Cap`). Rename `WorkspaceEdit` edits def + all
  references (incl. the clause-list and test-unit ones), dotted-segments only; **a colliding
  rename is refused by re-analysis**, **a capturing rename** (a use sits in a scope with a
  same-named local) **is refused by the index-unchanged-modulo-rename check**, and a
  **`prepareRename` on a local** returns null.
- **LSP query logic** *(unit-tested, no transport):* the index→`references` and index→rename-edit
  functions as **pure functions**; versioned-edit construction. This is likely the increment that
  **stands up the minimal `karn-lsp` integration harness** (v0.24 deferred it to the first feature
  needing round-trip testing — this is it).

## Diagnostics

None new in the language. Rename surfaces **request-time** failures (invalid new name; the
re-analysis validator found a regression) as LSP rename errors, not `karn.*` diagnostics.

## Risks
- **Sink completeness across the three sites** (`resolver.rs` + `checker.rs` + `project.rs`) is
  dominant — a missed site/kind silently corrupts a rename. Mitigate with the **kind-by-kind
  fixture matrix** (incl. static-receiver type mentions and checker-resolved capability/service
  op-calls) *and* the **re-analysis + index-equality validators** as backstops (a rename that
  breaks resolution **or** silently re-binds is refused, not applied).
- **Test/integration units** must be included (do **not** inherit `symbols.rs:28`'s skip).
- **Name-segment spans** for dotted/flattened/aliased forms (edit `Cap`, not `B.`).
- **Stale edits** → versioned `TextDocumentEdit`s.
- **Retaining resolution** for the index costs memory/CPU on the project pass; acceptable at
  current scale (debounced); incremental later.

## New decision records this increment adds

- **0053** — the LSP **binding index**: a **reference-table sink** (the `ErrorSink` analogue)
  threaded through **all three resolution sites** (`resolver.rs` + `checker.rs` + `project.rs`),
  recording **in-scope kinds only** (methods/fields deferred — the *kinds*, not the checker
  component), assembled and **unit-qualified at the v0.24 project-pass**; symbol identity is the
  **structural key** `(unit, kind, name)`; `references`/`rename` and a binding-correct
  `definition`+`hover` re-point consume it; **rename is validated by re-analysis (collisions) +
  index-unchanged-modulo-rename (capture/escape)**, `prepareRename` refuses out-of-scope symbols,
  and edits are versioned. *(Final numbering at implementation.)*

## Out of scope (→ later)

- **Methods, record fields, and op names** (checker-resolved / owner-qualified; ops also need
  provider-implementation edges) and **local-binding** references/rename → the next same-area
  follow-up.
- **Unit/context rename** (implies a file move) → A-3 **file operations**.
- **Workspace symbols** and **document highlights** → index-derived, cheap follow-ups (A-2).
- **Call hierarchy**, completion, signature help, inlay hints, semantic tokens.
- **Code actions** — still queued behind the A-0 work; returns on the project-diagnostic surface
  with the prescriptive `given`-clause catalogue.
