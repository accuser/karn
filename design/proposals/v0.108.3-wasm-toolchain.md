## What this adds (against the current code)

1. **`bynk_emit::project::compile_in_memory(source, target, platform)`** — runs the
   full project pipeline over one in-memory source, fs-free. The seam: `run_checks`
   takes an optional pre-discovered file list (skips `phase_discovery`); the source
   rides the existing `overlay`; the logical path is derived from the declared unit
   name so name↔path alignment passes. Returns the complete `ProjectOutput` (user unit
   + `runtime.ts` + `bynk-<platform>.ts` + `compose.ts`). `finish_build` is extracted
   and shared with `compile_project` (no behaviour change for the CLI/LSP).
2. **`strip_project_to_js` moved `bynkc` → `bynk-strip`** so the CLI and the wasm entry
   share one implementation (`bynk-strip` gains a `bynk-emit` dep; no cycle — the LSP
   still never pulls oxc).
3. **`bynk-wasm` crate (`publish = false`)** — one entry `bynk_compile(source) → JSON
   { files: [{path, contents}], diagnostics: [{path, line, col, severity, category,
   message}] }`, composing `compile_in_memory` (Bundle/Browser) with
   `strip_project_to_js`. The wasm-bindgen dep is `wasm32`-only; the native `rlib` is
   exercised by tests.
4. **A `wasm32` build-gate CI leg** building `bynk-wasm` for `wasm32-unknown-unknown`.

## Decisions

- **[DECISION A] One in-memory seam, reusing the whole pipeline** (not a
  hand-reimplemented subset) — so first-party injection, the binding, and the
  strip-only emitter behave identically to an on-disk build.
- **[DECISION B] `bynk-wasm` is `publish = false`** — a build artefact, not a library;
  sidesteps the new-published-crate process and keeps wasm deps out of native crates.
- **[DECISION C] Return the full stripped module graph** (the slice-4 API contract) —
  self-contained; the REPL links/caches as it likes.
- **[DECISION D] Ship `bynk-check` (Q3)** — full diagnostics; the whole pipeline
  compiles to `wasm32` cleanly (oxc + ariadne, no feature-gating). Size squeezing is a
  measured REPL-slice concern.
- **[DECISION E] Verify by native tests + a wasm32 build-gate**; in-browser execution
  proof lands with the REPL shell (slice 4).
- **[DECISION F] Version — patch v0.108.3.**

## Risks & mitigations

- **wasm payload size.** Real and unmeasured here; the REPL slice owns the budget
  (`wasm-opt`/thin-LTO/lazy-load), with an emit-only fallback if catastrophic.
- **No in-browser execution proof yet.** By design (verification depth): native tests
  + build-gate now; the browser/Worker harness is slice 4.
- **Long wasm compile.** oxc compiles for `wasm32` in minutes; cached after. The CI leg
  uses the shared rust-cache.
