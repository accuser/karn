# v0.28 — LSP semantic tokens (type-aware highlighting)

- **Status:** Draft (proposal). **Merging is approval to build.** *(A-2 — resolution-aware colouring
  beyond the client's syntactic layer. `karn-lsp` + a **medium** `karnc` index addition.)*
- **Builds on (verified):**
  - The cached **`ProjectIndex`** (v0.25, retained in `Analysis`) records every occurrence of
    **user-defined** symbols: `SiteRef { path, span }` (`index.rs:163`), `SymbolEntry { def, refs }`,
    keyed by `SymbolKey.kind ∈ {Type, Fn, Capability, Service, Agent, Provider}`. `RefEdge.span` is
    the **name-segment** span (`index.rs:62`), name segments never overlap — so no dotted/multi-token
    issue, and `def`-vs-`ref` gives the `declaration` modifier for free.
  - **BUT first-party (`karn.*`) symbols are *excluded* from the index.** Synthetic units are skipped
    at assembly (`project.rs:436`), and `IndexBuilder::build` **drops** edges whose target isn't a
    located user def (`continue; // builtin / first-party / unresolved target`, `index.rs:319`) then
    `index.symbols.retain(|_, e| e.def.is_some())` (`index.rs:352`). So references to `Kv`, `Clock`,
    `Fetch`, `Secrets`, the `karn.list`/`karn.map` fns — in `given`/capability calls/`consumes` — are
    **discarded**. The cached index does **not** "record every occurrence"; the platform-native
    payoff and *all `karn.*` colouring* need a side table (the decision below). *(This is the
    blocking correction from review: the v0.25 exclusion is deliberate — synthetic defs point at
    files not on disk, which `definition`/`rename`/`workspace-symbol` must never surface.)*
  - Modifiers aren't in the index: refined/opaque come from `TypeDecl` — and **`type Age = Int`
    parses as `Refined { refinement: None }`** (a plain alias; only `Some` is *refined*),
    `Opaque { refinement }` is independent (`ast.rs:660-676`). Platform-native is
    `firstparty::platform_of(unit)` (`firstparty.rs:78`).
  - `index_queries.rs` is the **pure-function home**; `tower-lsp` 0.20 has the `semanticTokens`
    full/range traits. No `semanticTokensProvider` advertised. Spans → analysed snapshot (v0.24).
- **Spec sections touched:** `design/karn-lsp-spec.md` (Semantic tokens + the legend/capability; §4.3);
  `karnc` `index.rs` (record modifiers on user symbols + a first-party `foreign_refs` side table).
- **Scope:** type-aware tokens for the index kinds (user-defined **and** first-party) with
  **declaration / refined / opaque / platform-native** modifiers, **additive over the client's
  syntactic layer**. **Not** here: locals / params / **generic type parameters** (deferred), the
  `delta` variant, keywords/literals (the syntactic layer owns those).

## Settled (recorded as ADRs ≥0057)

- **▸ Tokens surface the index's classification — a pure read of two sources, additive.** A new
  `index_queries` function reads the cached `ProjectIndex` for a file from **(a)** `symbols`
  (user-defined defs+refs) and **(b)** a new `foreign_refs` side table (first-party references), and
  emits tokens. The checker/resolver already classified; tree-sitter / the extension's TextMate
  grammar keeps owning keywords, literals, comments. Semantic tokens classify only **resolved
  identifiers** the syntactic layer can't tell apart; the client merges.
- **▸ Positions via the analysed snapshot; empty when no round is cached.** A freebie worth stating
  normatively: `test`/`integration` files' references *are* in the index (namespace-targeted at the
  defining unit), so **semantic tokens light up test files too**.

## Decisions (open — settle at review)

**[DECISION] First-party colouring via a token-only side table (the blocking fix; re-sizes the
`karnc` delta to *medium*).** Do **not** un-exclude synthetic units — that would leak nonexistent
def sites into `definition`/`rename`/`workspace-symbol`. *Rec:* in the builder, walk synthetic units'
decls into a `first_party_defs` map (kind + unit; **no usable def site**); add a **second
qualification pass** that routes the edges currently dropped at `index.rs:319` (target = first-party)
into **`ProjectIndex.foreign_refs: Vec<(SiteRef, SymbolKind, Modifiers)>`**, read **only** by the
tokens producer. `platform_of(unit)` is in hand at exactly that point, so `platformNative` lands
there; first-party refined types (e.g. the `karn` surface's `Uuid`) get `refined` the same way. **All
v0.25 invariants on `symbols` are untouched.**

**[DECISION] Record the modifiers on user symbols.** *Rec:* at assemble time (where the `TypeDecl` is
in hand) record on the `SymbolEntry`: **`refined` only when `refinement.is_some()`** (`type Age = Int`
= `Refined{None}` is a plain alias → **neither modifier**), **`opaque`** independently, and they are
**orthogonal** — `type T = opaque Int where …` gets **both** (bitsets are free).

**[DECISION] The legend — Karn kind → token type + modifiers.** The legend's **array order (token
types *and* modifiers) is the wire encoding** — declared in the capability and frozen; new entries
**append**, never reorder. *Rec:* standard token types where faithful — `Type → type`, `Fn →
function`; **custom** for the Karn-distinctive kinds — `Capability → capability`, `Service →
service`, `Agent → agent`, `Provider → provider` (custom types need theme defaults shipped by the
extension — a B-1 item; unthemed clients fall back to the syntactic colour). Modifiers:
`declaration`, `refined`, `opaque`, `platformNative`.

**[DECISION] `full` (+ `range`); `delta` deferred.** *Rec:* `textDocument/semanticTokens/full` (+
`…/range`); the **`delta`** variant is a later optimisation. Tokens are **delta-encoded** (relative
line/char, position-sorted) — a fixture pins the encoding.

**[DECISION] Coverage — index kinds now; locals + generic params deferred.** Variables, params,
lambda params, **generic type parameters** aren't in the index — a follow-up (resolver scopes / a
token harvest). Their identifiers keep the syntactic colour meanwhile (additive).

## Tests (the LSP analogue of fixtures)

- **`karnc`** *(`cargo test`, no LSP):* a **first-party reference** — `consumes cloudflare { Kv }` /
  `Kv.get(…)` — lands in `foreign_refs` with `SymbolKind::Capability` + **`platformNative`** (the
  corrected payoff test, now from the side table, not `symbols`); a **refined** user type carries
  `refined`, an **opaque** one `opaque`, a `opaque … where` carries **both**, and a **plain alias
  (`type Age = Int`) carries neither**; user symbols' `symbols` entries are unchanged (v0.25 tests
  still green).
- **`karn-lsp` pure-function level** *(the v0.25 harness — no transport):* the producer turns the
  cached index (`symbols` + `foreign_refs`) + a file into the expected **delta-encoded** array
  (encoding fixture: relative, sorted, correct type/modifier bitsets); a **legend-stability** test
  pins the order of **both** token types and modifiers; no-cached-round / out-of-file → empty. The
  advert is a **trivial unit check** — no transport claimed.

## Diagnostics

None new.

## Risks
- **First-party side table** — the medium part: a second qualification pass + a new `ProjectIndex`
  field; must not perturb the `symbols` invariants v0.25/v0.26/v0.27 rely on (references, rename,
  hints). Tokens-only readership keeps it isolated.
- **Legend stability** — token-type *and* modifier order is the encoding; frozen + a stability test;
  append-only.
- **Delta-encoding correctness** — relative, position-sorted; an off-by-one garbles colours.
- **Refined over-application** — gate on `refinement.is_some()`; the plain-alias fixture guards it.

## New decision records this increment adds

- **0057** — LSP **semantic tokens** read the cached index (`SymbolKind` → frozen legend), **additive
  over the client's syntactic layer**; **first-party references** are coloured via a tokens-only
  `foreign_refs` side table (synthetic units stay out of `symbols`, preserving the v0.25 navigation
  invariants); `refined`(only `Some`)/`opaque`/`platformNative`/`declaration` modifiers recorded at

  implementation, after v0.27's 0056.)*

## Out of scope (→ later)
