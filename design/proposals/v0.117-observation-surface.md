# v0.117 — the observation surface: auto-recorded calls at the capability seam, a thin `expect Cap.op called …` sugar, and a `trace(Cap.op)` escape hatch (testing track, slice 5)

- **Scope:** a single-increment **language + test-runtime** change — the sixth testing-track
  slice, the *interaction* rung. It adds, in a `case` body, observation of a consumed
  capability's calls: a thin sugar on `expect` — `Cap.op called` / `never called` /
  `called once` / `called <n> times` / `called … with <pred>` / `A.op before B.op` — plus a
  bound-trace escape hatch, `trace(Cap.op)`, which yields the recorded calls as an ordinary
  `List` asserted with ordinary Bynk. Calls are **recorded automatically** at the injected
  seam, in the test build only, so observation needs no setup. Version → **v0.117**.
- **Realises:** the testing track's *observation, not spies* pillar
  (`design/tracks/testing.md`) — because a capability is injected at a known seam, the
  runtime records its calls for free; a pure-observation `case` supplies nothing. The sugar
  covers the common facts (presence, count, argument shape, order); everything else is
  `trace(Cap.op)` + the existing value surface, so the observation vocabulary **never has to
  grow** to stay expressive.
- **Relates:** ADR 0144 (one predicate surface — a `with <pred>` reuses the invariant
  predicate over the call's arguments; no matcher library); the capability seam
  (`consumes` / `given` / a provider) — observation records at the *same* seam a test
  substitutes at, so it composes with today's `mocks` doubles and, after slice 6, with
  `provides` and real collaborators unchanged; slice 1a's `expect` and its structural
  failure reporting (an observation failure renders the recorded effect trace); the
  contract guard's build-profile discipline (ADR 0150 — recording is emitted in `bynkc
  test` only, stripped from the deploy build).
- **ADR at implementation:** one, numbered at authoring (next free after 0151; do not
  pre-allocate): **F** — *observation is auto-recorded at the seam, with a thin sugar and a
  `trace()` escape hatch.* Cites ADR 0144.

## The surface

Inside a `case`, a capability operation the unit under test consumes can be observed. The
subject is a **`Cap.op` reference** — the capability and one of its operations, *named, not
called* (no argument list):

```bynk
suite orders {
  case "an oversized order is rejected and logged" {
    let r <- Orders(Val[AcctId]).place(50000)
    expect r is Err(_)
    expect Logger.log called once with msg is "rejected: amount too large"
    expect Store.put never called            -- a rejected order writes nothing
  }
}
```

The **sugar** is a set of `expect`-clause forms over a `Cap.op` subject:

- **`expect Cap.op called`** — at least one call.
- **`expect Cap.op never called`** — zero calls.
- **`expect Cap.op called once`** / **`called <n> times`** — an exact count (`<n>` a literal).
- **`expect Cap.op called … with <pred>`** — at least one call whose arguments satisfy the
  predicate. The predicate is the **invariant predicate** over the operation's parameters,
  in scope by their declared names (`with msg is "…"`, `with amount > 1000`); it is pure
  `Bool` — no effects, capabilities, `expect`, or `Val`. `with` composes with a count
  (`called once with …` = exactly one call, and it matches).
- **`expect A.op before B.op`** — an ordering claim: the first `A.op` call happens before
  the first `B.op` call (both must have occurred).

The **escape hatch** — for anything the sugar does not cover — binds the recorded calls as
an ordinary value and asserts over it with the existing surface:

```bynk
let calls = trace(Logger.log)
expect calls.length() == 2
expect calls.all((c) => c.msg.length() > 0)
```

`trace(Cap.op)` yields a **`List[<CallRecord>]`** in call order, where `<CallRecord>` is a
synthetic record of the operation's parameters (`{ msg: String }` for `Logger.log`). It is
an ordinary `List`, so it is asserted with the methods that already exist — `length()`,
`all`/`any`, indexing — inside a single `expect`. This is the deliberate reuse that keeps
DECISION F's promise: **no test-only iteration construct is added** — a claim "for every
recorded call …" is `calls.all((c) => …)`, one predicate, not a second `for all` (see
DECISION F2).

## The mechanism — recording at the seam (DECISION F)

A capability call lowers to `<deps>.<Cap>.op(args)` — the injected seam. In the **test
build**, the runner wraps every capability operation reachable from the unit under test
with a **recording proxy**: each call appends its arguments (and a monotonic order index)
to a per-operation log, then delegates to whatever stands behind the seam — a `mocks`
double today, a `provides` stub or a real collaborator after slice 6. So:

- **Observation needs no setup.** A pure-observation `case` declares no `mocks`/`provides`;
  the recording is ambient at the seam. Overriding a *return* is a separate concern
  (`mocks` now, `provides` later) and only needed when the test depends on the value.
- **The sugar and `trace` read the same log.** `Cap.op called once with <pred>` and
  `trace(Cap.op)` are two views of one recorded list; they can never disagree.
- **Recording is test-only.** The proxy is emitted only under `bynkc test` (the same
  build-profile switch the contract guard uses, ADR 0150) — the deploy build calls the
  seam directly, so observation adds no production cost or behaviour.
- **A failure shows the trace.** A failed observation reports, beside the expected-vs-actual
  (slice 1a), the recorded effect trace for the case — the ledger of calls that did happen,
  in order — so "never called, but it was called once" points at the offending call.

Observation targets **capability operations** — the injected seams the unit `consumes` /
`given`s. Observing a `Cap.op` the unit does not consume is `bynk.observe.not_a_seam`
(mirroring `provides.not_a_seam`, Principle 4); a `Cap.op` naming no operation of the
capability is `bynk.observe.unknown_op`. Agent-handler and cross-context-wire observation
are named follow-ons (Out of scope).

## Decisions

**[DECISION F1] The `with` predicate binds the operation's parameters by name.** `with
<pred>` is the invariant predicate with the operation's parameters in scope by their
declared names (`Logger.log(msg: String)` → `msg` in scope), so `with msg is "…"` reads
directly and multi-argument operations need no positional indices. *Recommend* parameter-
names-in-scope (matches the track example and the one predicate surface); the alternative —
a single `call` record threaded as `with call.msg == …` — is noisier and is rejected. A
non-`Bool` `with` is `bynk.observe.with_not_bool`; an impure one `bynk.observe.impure_with`.

**[DECISION F2] The escape hatch reuses `List`; no test-only iteration is added.** The
track sketched `for all c in calls { expect … }`. That would be a *second* iteration
construct competing with `List.all`/`List.any`, and `for all` already means *generative*
binding (slice 2). *Recommend* `trace(Cap.op)` returns an ordinary `List[<CallRecord>]`
asserted with the existing List surface (`calls.all((c) => …)`, `calls.length()`,
indexing) — one predicate surface, no new grammar. A dedicated in-test `for all … in` is
**rejected**, not deferred.

**[DECISION F3] The sugar words are contextual, not reserved.** `called`, `never`, `once`,
`times`, `with`, `before` are parsed only in the observation-clause position after a
`Cap.op` subject in an `expect`; everywhere else they stay ordinary identifiers (as
`old`/`new`/`result`/`for`/`all` are). *Recommend* contextual — reserving `with`/`before`
as keywords would break existing identifiers for no gain. `trace` is a **test-only
builtin** (like `Val`): `trace` outside a `case` is `bynk.observe.trace_outside_test`.

**[DECISION F4] `before` is first-before-first.** `A.op before B.op` holds iff both
occurred and the *first* `A.op` precedes the *first* `B.op`. *Recommend* this simple,
predictable rule; richer temporal orderings (every-A-before-every-B, interleavings) are a
`trace`-plus-`List` concern, not sugar. Both operands must have been called, else the claim
fails with a trace.

**[DECISION F5] The call-record type is synthetic, per operation.** `trace(Cap.op)` yields
`List[<CallRecord>]` where `<CallRecord>` is a record whose fields are the operation's
parameters at their declared types — so `c.msg` type-checks like any record access. It is a
value type (ordinary field access, `is`, methods); it is test-only, produced only by
`trace`.

## What changes, against the current code

The observation forms and `trace` are new; the predicate checking, the `List` surface, and
the capability-seam lowering are reuse.

1. **Lexer / keywords** — no new hard keywords: `called`/`never`/`once`/`times`/`with`/
   `before` are contextual in the observation clause; `trace` is a contextual test-only
   builtin (an identifier elsewhere).
2. **AST / parser** — an `expect` gains an **observation-clause** alternative: a `Cap.op`
   subject (a capability-op reference, not a call) followed by one of the sugar forms
   (`called` [`once` | `<n> times`]? [`with` `<pred>`]?, `never called`, `before Cap.op`).
   A `trace(Cap.op)` primary expression. Reuse the predicate / `is` / `implies` parsers for
   `with`.
3. **Checker** — resolve `Cap.op` against the unit's consumed capabilities (the
   `consumes`/`given` graph the checker already builds); reject a non-seam
   (`bynk.observe.not_a_seam`), an unknown op (`bynk.observe.unknown_op`), a non-`Bool` /
   impure `with` (`bynk.observe.with_not_bool` / `impure_with`), a non-literal or negative
   count (`bynk.observe.bad_count`), and `trace` outside a `case`
   (`bynk.observe.trace_outside_test`). Type `with`'s predicate scope = the op's parameters;
   type `trace(Cap.op)` as `List[<CallRecord>]` with a synthetic record registered like the
   agent state record (slice 4 precedent).
4. **Emitter — recording proxy** (`bynk-emit/src/project/tests_emit.rs`) — in the test
   build, wrap each observable capability operation on the case's `deps` object with a
   proxy that records `{args, order}` then delegates. Lower the sugar to reads over that
   log (`called once with` → count + a `some(...)` over the recorded args using the lowered
   `with` predicate; `before` → index comparison); lower `trace(Cap.op)` to the recorded
   `List`. Fold a failed observation into the slice-1a expected-vs-actual renderer plus the
   effect-trace dump. Emitted only under the `bynkc test` profile (ADR 0150 switch).
5. **Diagnostics** — the `bynk.observe.*` family and an "Observation" category.
6. **tree-sitter** — an `observation_expr` (the `expect Cap.op …` clause) and a
   `trace_expr` on the expression grammar; `called`/`never`/`with`/`before` as contextual
   highlights; corpus + `highlights.scm`; re-bless `grammar.json` and the embeddable-rule
   count. **vscode-bynk** — highlight the contextual observation words in the test context.

## Migration (clean-slate; additive)

Observation is **new surface** — nothing is removed or renamed, so no codemod is needed.
`mocks` / `given` are unchanged this slice (the seam they establish is exactly what
recording hooks into; slice 6 later renames `mocks` → `provides` with no effect on the
observation vocabulary). New fixtures cover: `called` / `never called` (pass and fail with a
trace dump), `called once` / `called <n> times`, `with <pred>` matching and not, `before`
ordering, a `trace(Cap.op)` + `List.all` escape-hatch assertion, a `not_a_seam` rejection,
an `unknown_op` rejection, an impure/non-`Bool` `with`, and `trace` outside a `case`. A
behaviour fixture drives a recorded case under Node and asserts the sugar and `trace` agree.

## Risks

- **Recording fidelity vs the value form.** The proxy must record the *arguments as passed*
  (post-evaluation) without perturbing them, and delegate transparently so a `mocks`/real
  return is unchanged. A behaviour fixture asserts the recorded args equal the passed args
  and the delegated return is untouched.
- **Contextual sugar words.** `with`/`before`/`called` must stay usable as ordinary
  identifiers outside the observation clause; parse them contextually (as `old`/`new` in
  slice 4) and verify with a fixture binding values named `with`/`before`.
- **`with` predicate purity.** The `with` predicate is the one predicate surface and must
  be pure — no effects or capability calls inside an observation predicate
  (`bynk.observe.impure_with`), same posture as every other predicate position.
- **Test-only strip.** Recording must be *provably* absent from the deploy build (a drift-
  guarded fixture asserts the deploy emit wraps no capability op), so observation never adds
  production cost — the same guarantee the contract guard gives (ADR 0150).
- **Ordering semantics must be stated.** `before` is first-before-first (DECISION F4);
  documented so a reader is never surprised, with richer orders left to `trace` + `List`.

## Out of scope (named, deferred to their slices)

- **Agent-handler and cross-context-wire observation.** v1 observes capability operations
  at the in-process seam; observing agent handlers, or calls across the real serialise →
  JSON → deserialise boundary (the `system` tier), is a named follow-on.
- **The tier dial + `provides`** (retires `mocks`) — slice 6 (DECISIONS G/M/R). Observation
  records at whatever seam the tier establishes, unchanged.
- **The universal-emission guarantee** ("every payment audits, on every path") — not a
  per-case observation at all; a cross-cutting policy / use-case minimal guarantee, deferred
  (DECISION E/U).
- **History / handler-sequence properties** — slice 7 (DECISION H).
