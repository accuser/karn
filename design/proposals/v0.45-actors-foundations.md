  review. This slice builds the whole boundary-contract machine against the two zero-crypto schemes;
  external auth, refinement invariants, and multi-actor sums are later slices.)*
- **Sequences after v0.44** (`service from <protocol>`). The `by` clause attaches to v0.44's
  header-relative handler forms, and per-protocol default actors key off the `from <protocol>`
  header. **Build-gated on v0.44 landing first**; the version is provisional (`v0.45` assumes v0.44

- **Lands the foundational ADRs** for Q1/Q2/Q5 (numbers assigned at landing, after v0.44's).

## Context — make the boundary contract real, against trivial schemes first

Today a handler's actor is implicit and untyped: cron is "the platform Scheduler," an HTTP route is
"whoever called," and the runtime trusts the channel. The design notes (§6/§7) and the actors track
make the actor a **typed contract** consumed on a `by` clause — the boundary verifies it, mints a
typed identity, and the body runs only if it holds. This slice builds that entire machine —
declaration, `by` clause, identity binding + threading, contract-satisfaction checking, the
verification seam, and per-protocol default actors — but admits only the two **zero-crypto** schemes
(`None`, `Internal`). Every later slice is then "add a scheme/feature," not a re-architecture. The
felt value now: the existing protocols' actors become explicit and typed (`Scheduler`, the queue
producer, the event bus, the cross-context caller are all `Internal`; a public HTTP route declares
`Visitor`), and the seam + descriptor are in place for Bearer/Signature to slot into.

## Settled (from the reviewed design doc)

- **▸ `actor <Name> { auth = <Scheme> }` (optionally `, identity = <T>`) is a nominal contract type**,
  not a runnable entity (Q1). The scheme set is **closed and compiler-known**; this slice admits
  `None` and `Internal` only. A `None` actor yields `identity = ()`; an `Internal` actor yields the
  caller identity its channel supplies (e.g. the calling context).
- **▸ The `by <name>: <Actor>` clause sits on the handler, after the protocol config and before the
  parameters** (Q5): `on schedule("…") by s: Scheduler () -> …`. The verified actor binds to
  `<name>`; its identity is `<name>.identity`. Verification is **two-phase**: it must succeed before
  the body runs, with the payload already parsed (fail-closed — no verified identity ⇒ no body).
- **▸ The verified identity is a context-sealed value** (Q2): minted only by the verification seam,
  flowing service → agent unchanged, **never re-checked downstream**. Agents receive an
  already-authoritative identity.
- **▸ Per-protocol default actors** (Q5): with the protocol on the `from` header, a handler that
  omits `by` inherits the protocol's default — cron → `Scheduler`, queue → its producer, events →
  the bus, `on call` → the calling context (all `Internal`). **HTTP has no safe default, so `by` is
  required on HTTP handlers** — a public route writes `by v: Visitor` explicitly. There is no
  implicit anonymous surface.
- **▸ The verification seam is a sealed scheme descriptor** mirroring v0.44's protocol descriptor:
  each scheme contributes its verification codegen, identity shape, and failure mapping behind one
  interface. `None` emits nothing (always admits); `Internal` emits the channel-trust assertion
  already implicit in service-binding/platform dispatch. Adding Bearer/Signature later is new surface
  against this interface.

## Decisions (open — settle at review)

**[DECISION] Does cross-context `Internal` identity (Q7) fold into Foundations?** With only
`Internal`/`None`, the only non-trivial identity available is the **calling context's** identity on
an `on call` / cross-context handler. *Rec:* **fold in the read-only calling-context identity** — it
is the one place Foundations carries a *useful* identity (otherwise every Foundations identity is `()`
or a bare `Scheduler` tag), it exercises the threading end-to-end, and it reuses the v0.6 binding
channel. *Alternative:* keep Foundations to declaration+seam only and defer all identity payload to
the Bearer slice — smaller, but ships a machine with nothing meaningful flowing through it.

**[DECISION] Identity access surface: `name.identity` vs `name` *is* the identity.** *Rec:*
`name.identity` — the binding is the *verified actor* (it may later carry claims, scheme metadata),
and `.identity` is its identity value; this leaves room for `name.scheme` / claim accessors without a
surface change. *Alternative:* bind the identity directly (`by uid: User` gives `uid: UserId`) —
terser, but conflates the party with its identity and forecloses actor-level accessors.

**[DECISION] The refinement form `actor Admin = User where …` (Q3) — reserve the grammar now or
later?** *Rec:* **reserve and reject** in Foundations (`karn.actor.refinement_unsupported`) so the
invariants slice adds *admission*, not grammar — mirrors v0.44's "reserve the `protocol` keyword".
*Alternative:* don't parse it yet — less surface now, a grammar change later.

**[DECISION] Default-actor surface — silent inheritance vs an explicit marker.** *Rec:* **silent**
(omitting `by` inherits the protocol default) — the default is the architectural norm for
`Internal` protocols and demanding `by s: Scheduler` on every cron handler is ceremony.
*Alternative:* require `by` everywhere for symmetry — more uniform, more boilerplate on the common
case.

## Spec delta

- **`syntactic-grammar.md`.** New `actor_decl` (`actor` name, `{ auth = scheme (, identity = type)? }`;
  the `= base where predicate` refinement form reserved-and-rejected per the decision). New optional
  `by_clause` on the handler productions (`by <name>: <actor_ref>`), positioned after the protocol
  config and before parameters. A `scheme` production over the closed token set (`None`, `Internal`
  admitted; `Bearer`, `Signature` reserved-and-rejected).
- **`static-semantics.md`.** New rules: actor declaration well-formedness (scheme in the admitted
  set; identity type is a context-ownable value type); the `by`-clause contract-satisfaction rule;
  per-protocol default-actor resolution and the admissible-scheme-per-protocol check; HTTP-requires-
  `by`; the sealed-identity rule (a verified identity is minted only at the seam, is non-forgeable,
  and is not re-verified); fail-closed verification ordering (verify before body).
- **`runtime-library.md`.** The compiler-known default actors (`Scheduler`, the queue producer, the
  event bus) and `Visitor` as prelude actors; `None`-actor identity is `()`. No new generic types —
  identity types are user-declared value types.
- **`emission.md`.** The scheme descriptor and the per-protocol verification seam: `None` → admit;
  `Internal` → the channel-trust assertion already emitted for service-binding/platform dispatch,
  now gated and fail-closed; identity construction at the seam. No target-topology change beyond the
  gate (the Worker entry points are unchanged; verification wraps the existing dispatch).
- **`diagnostics.md`.** New `karn.actor.*` family: `unknown_scheme` (with the
  Bearer/Signature-not-yet fix-it), `scheme_unsupported`, `refinement_unsupported`,
  `missing_by_on_http`, `unknown_default_actor`, `identity_not_sealed`, plus the contract-mismatch
  codes.
- **`lexical-grammar.md`.** `actor` / `by` move from reserved to productive; reserve scheme names
  (`None`, `Internal`, `Bearer`, `Signature`) in actor-header position.
- **LSP / tooling.** `actor` declarations and the `by` clause flow through the binding/member
  indices, semantic tokens (likely a new `actor` token kind — check the legend-drift guard,
  ADR 0058), hover, and document symbols; `identity` access participates in member completion.
  Scope these now per the cross-component drift guard.

## Tests

- **Positive fixtures:** an `actor` decl of each admitted scheme; `by` on cron/queue/events/call
  (default and explicit); a public HTTP route with `by v: Visitor`; identity threading service →
  agent; the calling-context identity (if Q7 folds in).
- **Negative fixtures:** `auth = Bearer` (scheme-unsupported, with fix-it); a refinement
  `actor Admin = User where …` (refinement-unsupported); an HTTP handler with no `by`
  (`missing_by_on_http`); an identity type that isn't context-ownable (`identity_not_sealed`); a
  handler body that reaches identity before verification can't be expressed (fail-closed is
  structural).
- **Golden emission:** the verification seam for `None`/`Internal`; prove the gate is fail-closed and
  that `Internal` reuses the existing service-binding trust assertion (diff is the gate + identity
  mint, not new topology).
- **LSP:** semantic-token + binding-index + hover fixtures over a `by`-header service; legend-drift
  guard green.

## Diagnostics / Risks

- **v0.44 coupling.** This slice's grammar and per-protocol defaults are written against v0.44's
  header-relative handler forms. If v0.44's surface moves in review, the `by`-clause position and the
  default-actor mechanism move with it. Mitigation: the proposal is build-gated on v0.44; implement
  only once v0.44's grammar is fixed.
- **Sealed-identity ergonomics.** Requiring identity types to be context-ownable may surprise authors
  who reach for a bare record. Mitigation: the `identity_not_sealed` diagnostic explains the rule and
  points at the context-construction seal.
- **A machine with trivial payloads.** With only `None`/`Internal`, identities are `()` or channel
  tags unless Q7 folds in. Mitigation: the calling-context-identity decision; otherwise the slice is
  explicitly machinery-first and the Bearer slice delivers the first authenticated identity.

## New decision records this increment adds

*(Numbers assigned at landing, after v0.44's ADRs.)*

- **ADR (Q1) — Authentication schemes are a closed, compiler-known nominal set; actors are nominal
  contract types.** Sealed-now-openable-later (widen the enum); Foundations admits `None`/`Internal`.
- **ADR (Q2) — A verified actor identity is a context-sealed value.** Minted only at the boundary,
  flows service→agent, never re-checked; not a plain record, not linear.
- **ADR (Q5) — The `by` clause; two-phase verify-then-body; per-protocol default actors.** `by` on
  the handler, identity as `name.identity`, HTTP requires explicit `by`.
