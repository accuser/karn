# v0.76 — Capability calls legible in the stack: Bynk operation names via a debug-metadata sidecar (semantic-debugging track, slice 3)

- **Track:** [semantic-debugging](../tracks/semantic-debugging.md), slice 3. Slice 2 made
  the handler *frame's variables* read in Bynk; this makes the *call stack* read in Bynk
  — handler frames named for their Bynk operation, not the emitted JS function.
- **Realises:** ADR [0105](../decisions/0105-semantic-debug-interposition.md) — the same
  editor-side interposer, now rewriting the `stackTrace` response. **This is the slice
  that introduces the emitter debug-metadata sidecar D5 anticipated** — so it also
  discharges slice 2's deferral (the `by` actor) and hardens slice 2's name-inference.
- **Posture:** tooling + a **first emitter touch** (a sibling debug-metadata artifact; no
  language surface, no change to the emitted `.ts`).

## What Phase 1 already gives us (the grounding)

A real stack capture (breakpoint in a handler, `skipFiles` set as the provider ships it)
shows the legibility is *already half-done*:

```
FRAME name="http_GET"  src=".../svc.bynk"            hint="normal"        ← the user's handler
FRAME name="dispatch"  src=".../runtime.ts"          hint="deemphasize"   ← glue, already greyed
FRAME ... <node_internals> ...                        hint="deemphasize"   ← glue, already greyed
```

So **"toolchain/glue frames collapse" is already done** — js-debug deemphasises every
`skipFiles` frame (Phase 1). And the user's frame already *points at its `.bynk` line*
(source-mapped). The single remaining gap is the frame **name**: it reads `http_GET` (the
emitted JS function), not the Bynk operation (`GET "/"`).

## Why this needs metadata (the D5 line, finally crossed)

The frame name `http_GET` carries the *verb* (`GET`) but **not the route** (`"/"`), and an
agent method (`increment`) carries no signature. The readable label — `GET "/"`,
`add(n: Int)`, `on call increment()` — **isn't in the emitted name**; only the compiler
knows it. Inference would give a thin `http_GET → GET` and stop there. So slice 3 is the
first feature whose value genuinely requires the **emitter debug-metadata sidecar** that
slices 2–3 named and deferred — and it is the right forcing function to design it well,
once, for the stack *and* the slice-2 actor.

## The sidecar — minimal, and shared

`bynk-emit` emits a sibling **`<file>.bynkdbg.json`** next to each `.ts`/`.ts.map`
(profile-gated exactly like `sourcesContent`, [[0103]] D6 — never in a deployed bundle):
a small map from each emitted handler function to its **Bynk operation label** (and, for
the deferred items, which parameter is the `by` actor). The interposer loads it per
session (keyed by the frame's source path), so the rewrite stays editor-side and
runtime-agnostic. The map is tiny and additive; the emitted `.ts` is unchanged.

## What this slice does

1. **Emit the sidecar** (`bynk-emit`): for each handler, record `{ fn → label }` where
   `label` is the Bynk operation (`GET "/"`, `add(n: Int)`, `on call increment()`,
   test-case names). Decode-golden tested, like the source maps.
2. **Relabel `stackTrace` frames** (the interposer, `src/debug.ts`): for a frame whose
   source is a `.bynk` file, look up its label in the sidecar and replace the frame
   `name`; leave glue frames (already `deemphasize`) and non-Bynk frames untouched.
3. **Verify the glue collapse** is complete for the workerd worker glue
   (`compose.ts`/`index.ts`/wrangler middleware) — extend `skipFiles` if the capture
   shows a gap; this is the "frames collapse" half, mostly already done.
4. **Fold in the slice-2 actor (stretch).** With the sidecar carrying the actor
   parameter, surface the `by` actor in the frame's variables (the slice-2 deferral) —
   if it lands cheaply; otherwise keep it a named follow-on.
5. **Tests:** a sidecar decode golden (`bynkc`-side) and a Node integration test (stop in
   a handler; assert the frame reads its Bynk operation); workerd local/opt-in. Slice
   0–2 guards stay green.
6. **Docs:** extend the "handler frame in Bynk" guide note to the call stack; changelog;
   the bump.

## Decisions

**[DECISION A — recommend] Introduce the debug-metadata sidecar here.** The stack relabel
is the first feature inference can't carry (the route/signature isn't in the emitted
name), so this is the principled place to add the sidecar — designed once for the stack
and the slice-2 actor, not bolted on later. *Recommend over the thin alternative* (relabel
`http_GET → GET` from the JS name only), which adds little over what source-mapped frames
already show.

**[DECISION B] Editor-side rewrite, per ADR 0105.** The relabel is a `stackTrace`-response
rewrite in the existing interposer; the sidecar is *data*, loaded editor-side. No new
mechanism, runtime-agnostic.

**[DECISION C] Sidecar is profile-gated and `.bynk`-source-keyed.** Emitted only for
debug/dev builds (never a deployed bundle, [[0103]] D6); the interposer keys lookups on
the frame's `.bynk` source path + the function, so it only ever relabels Bynk frames.

**[DECISION D] Glue collapse is already Phase 1's; this slice only verifies/extends it.**
`skipFiles` deemphasises glue today; no new collapse mechanism — just confirm coverage.

**[DECISION E] Version — minor v0.76.** New extension capability + an additive emitter
artifact; no language change, no emitted-`.ts` change.

## Risks & mitigations

- **Sidecar drift from the emitted code.** The label map must stay in lockstep with the
  function names it keys. Mitigated by emitting it from the same lowering that names the
  functions, and a decode golden that compiles a handler and checks the label.
- **Interposer ↔ sidecar coupling.** The editor-side loader must tolerate a missing/stale
  sidecar (fall back to the raw frame name — never break the stack). Total-by-default.
- **Scope creep via the actor (DECISION D/4).** Kept a *stretch*; the slice ships the
  stack relabel even if the actor slips to a follow-on.
- **`variablesReference`/frame identity.** Renaming a frame's `name` must not disturb its
  `id`/`source`/`line` (clicking still navigates). The rewrite touches only `name`.
