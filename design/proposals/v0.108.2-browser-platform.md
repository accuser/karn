# v0.108.2 — The Browser platform (in-browser track, slice 2)

- **Scope:** the third slice of the [in-browser track](../tracks/in-browser.md) —
  a **checker/emitter** increment adding `Platform::Browser` and its `bynk` surface
  binding. No grammar or server change. Versioned within the track's v0.108 minor
  (slice 0 = v0.108.0, slice 1 = v0.108.1, this = v0.108.2).
- **Realises:** the track's §3.2 (Layer 2 — the Browser platform) and its slice-2
  line in §6; takes the Q2 decision (Fetch withheld). Lands the **Browser platform
  ADR** ([ADR 0138](../decisions/0138-browser-platform.md)). **Security-bearing**
  (the binding is the playground's capability boundary) — feature-track posture.

## What this adds (against the current code)

1. **`Platform::Browser`** (`bynk-check/src/firstparty.rs`) — third deploy platform
   alongside Cloudflare/Node; `as_str() => "browser"`, binding filename/source.
2. **`bynk-browser.ts`** — the `bynk` surface over Web APIs. Clock/Random/Logger
   byte-identical to the Node binding; **`Fetch` withheld** (`send` throws) and
   **`Secrets` unavailable** (`get` throws) — fail loudly (Q2 / §4), no
   `FetchError.Unavailable` variant added.
3. **`--platform browser`** (`bynkc/src/cli.rs`, `CliPlatform::Browser`).
4. **Bundle-only enforcement** — `--platform browser --target workers` rejected up
   front with a new diagnostic `bynk.target.browser_bundle_only`
   (`bynk-emit/src/project/validate.rs`, registered in `bynk-syntax`).
5. **No new platform-lock code** — a browser build pulling in `bynk.cloudflare` is
   rejected at validate time (`bynk.target.vendor_required`) by the existing
   native-platform machinery; slice 2 just confirms and pins it.
6. **Fixtures + harness** — `fixture_platform` (tsc_verify + e2e) accepts
   `browser`; positive `239_bynk_browser_platform` (carries `bynk-browser.ts`
   through the tsc/strip/JS suites); negatives `267_browser_target_workers` and
   `268_cloudflare_on_browser`.

## Decisions

- **[DECISION A] Fetch withheld, Secrets unavailable, both throw** (Q2). A browser
  *can* fetch, but the playground binding refuses egress (SSRF/exfil); a browser has
  no secret store. Throwing fails loudly and educationally; no surface change.
- **[DECISION B] Browser is Bundle-only.** A browser cannot do the Workers wire
  model; the combination is a compile error, enforced at validate time.
- **[DECISION C] Reuse the existing platform lock** for Cloudflare-only units — no
  new code; the native-platform machinery already fires.
- **[DECISION D] Version — patch v0.108.2** (track-internal increment).

## Docs delta

## Risks & mitigations

- **Third-binding parity drift.** The capability surface is small and shared; the
  positive fixture runs the browser binding through `tsc --strict` + strip + JS.
- **Withheld Fetch surprises users.** The thrown error names the limitation and
  points at node/cloudflare; the playground UI will state the in-process scope.
- **Egress abuse.** Addressed by the default-withhold posture; revisit only behind a
  proxy (Q2 follow-on).
