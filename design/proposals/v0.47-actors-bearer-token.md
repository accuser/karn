# v0.47 — Actors, slice 2 (BearerToken): the first authenticated scheme — compiler-generated JWT verification, secret sourcing, and the 401 path

- **Status:** Draft (proposal). **Merging is approval to build.** *(Feature-track
  slice — the second of the actors track, `design/tracks/actors.md`. Slice 1
  (Foundations, v0.45) built the whole boundary-contract machine against the
  zero-crypto schemes; this slice adds the first **authenticated** scheme and the
  first real minted identity, slotting into the sealed scheme descriptor the
  Foundations seam left open.)*
- **Sequences after v0.45** (actors Foundations, landed — ADRs 0080/0081/0082).
  **Build-gated on v0.45** (it is in). The version is provisional (`v0.47`
  assumes the queued `v0.46-karn-doctor` lands first — renumber if sequencing
  shifts). The *design* is independent of the gate.
- **Verification-bearing slice.** This is the first slice that emits real
  verification logic, so it carries the `/security-review` + `/code-review` gate
  before landing (ADR 0076), and a threat model below.

## Context — turn the sealed seam into a real authenticator

Foundations admitted only `None` (anonymous) and `Internal` (channel trust);
both mint a unit identity and emit no verification code. The whole point of the
actors track is that **the compiler generates the boundary verification a service
would otherwise hand-write** (track throughline; Servant's `Auth '[JWT] User`
prior art). This slice cashes that in for the most common external case: a
**Bearer token** on an HTTP route. A handler writes `by u: User` where
`actor User { auth = Bearer(secret = "AUTH_JWT_SECRET"), identity = UserId }`,
and the compiler emits — at the boundary, before the body runs, fail-closed — the
code that extracts the `Authorization: Bearer …` header, verifies the token,
mints a typed `UserId`, and returns **401** on any failure. No app-written crypto.

## Builds on (verified)

- **The v0.45 scheme descriptor + `by` clause** (ADRs 0080/0081/0082). `Scheme::Bearer`
  is already a reserved enum variant (`karnc/src/actors.rs`), rejected today by
  `admitted()`; this slice flips it to admitted and gives it a descriptor entry.
  The `by` clause, the `Ty::Actor(identity)` binding, and `<binder>.identity`
  type-checking are all in place.
- **`HttpResult.Unauthorized` (401) and `HttpResult.Forbidden` (403)** already
  exist in the emitted runtime (`emitter.rs`, `httpResultToResponse`). 401 is the
  Bearer failure channel; 403 is reserved for the invariants slice (Q3).
- **The `Secrets` capability** (`capability Secrets { fn get(name: String) -> Effect[Option[String]] }`,
  `firstparty.rs`; ADR 0021) and its `SecretsProvider(env)` — explicit `env`
  first (workers compose passes it), then a `process.env` probe (bundle). The
  signing secret is sourced through exactly this, instantiated at the boundary
  from the `env` already in scope in the Worker entry **before** `compose(env)`.
- **The HTTP entry dispatch** (`emitter/workers_entry.rs`, `emit_http_route_dispatch`)
  — the `request` object and `env` are in scope at the dispatch site; the seam
  injects between path-param construction and the handler call.
- **String-constructible identity types** — the same `.of`-with-refinement
  machinery HTTP path params already use to turn a `String` segment into a
  branded value; the minted claim flows through it.

## Settled (from the reviewed design doc / foundational ADRs)

- **▸ Bearer is in the closed scheme set** (ADR 0080) — no new scheme *kind*, just
  the reserved `Bearer` variant becoming admitted. Adding it is new *surface*
  against the scheme descriptor, not a re-architecture.
- **▸ Verification is two-phase and fail-closed** (ADR 0082) — the token is
  verified before the body runs, with the payload already parsed; no verified
  identity ⇒ no body. This slice realises that for a non-trivial scheme.
- **▸ The identity is a context-sealed value** (ADR 0081) — minted only at the
  seam, flowing service → agent, never re-checked. This slice mints the **first
  real** such value and resolves the v0.45 `.identity`-lowering note (it lowered
  to `undefined`; it now lowers to the seam-bound value).
- **▸ The 401/403 split** (track Q6) — authentication-scheme failure → **401**;
  the 403 (a satisfied scheme but a failed authorisation *invariant*) is the
  refinement slice (Q3) and is **out of scope** here. Bearer only ever yields 401.

## Decisions (open — settle at review)

**[DECISION] Verification model: compiler-generated JWT/HS256.** *Rec:* the
compiler emits **JWT verification with HS256** (WebCrypto `crypto.subtle`,
HMAC-SHA256) against a secret sourced via `Secrets` — realising the track
throughline (the compiler generates the verification) and matching the Servant
JWT prior art. *Alternatives:* (a) a **user-supplied verifier fn** named on the
actor — maximally flexible but reintroduces the hand-written-crypto footgun the
feature exists to remove; (b) an **opaque shared-secret compare** (`token ==
secret`) — trivial, but yields no per-party identity, so it can't deliver the
"authenticated identity" this slice is for. *Scope:* HS256 (symmetric, one
shared secret) only; RS256/ES256 + JWKS and opaque-token-lookup are later slices.

**[DECISION] Identity source: the `sub` claim through the identity type.** *Rec:*
the minted identity is the JWT **`sub`** claim, a `String`, **constructed through
the declared `identity = T` type's string-constructor** (`.of` + refinement) —
so an absent/ill-formed `sub` fails closed → 401, and `T` reuses the existing
refinement machinery. Consequence: a Bearer actor's `identity` **must be a
string-constructible type** (a refined/opaque `String`), diagnosed otherwise.
*Alternative:* map the full claim set onto a **record** identity — richer, but
needs a claims-schema vocabulary; deferred (the declaration shape here does not
foreclose it).

**[DECISION] Secret naming: `auth = Bearer(secret = "ENV_NAME")`.** *Rec:* the
actor names the secret inline; the name is resolved by `Secrets` (env →
`process.env`). This extends `scheme` from a bare token to a **parameterised
form** (grammar delta below). *Alternatives:* a convention-derived name
(`<ACTOR>_JWT_SECRET` — implicit, fragile) or a required `given Secrets` + fixed
name (couples every Bearer handler to a `given`).

**[DECISION] Where verification runs and how it reaches the secret.** *Rec:* the
seam emits in the **HTTP entry dispatch**, before the handler call, instantiating
`SecretsProvider(env)` directly from the `env` already in scope (the boundary
runs before `compose`, so it does **not** go through handler `deps`). Bundle
target sources from `process.env`. Fail-closed → `HttpResult.Unauthorized`. This
keeps verification a boundary concern, not a body dependency.

**[DECISION] Bearer is HTTP-only.** *Rec:* `scheme_admissible` admits `Bearer`
only on `from http` (an `Authorization` header is an HTTP concept); on
call/cron/queue it is rejected (`karn.actor.scheme_not_admissible`, already
emitted). `Internal` remains the internal-protocol scheme.

**[DECISION] Expiry validation.** *Rec:* the seam validates the JWT **`exp`** (and
`nbf` when present) against `Date.now()` emitted inline — an expired/not-yet-valid
token → 401. No `Clock` capability is threaded (the seam is emitted TS and may
call `Date.now()` directly). *Alternative:* skip `exp` in slice 2 — rejected: a
security feature that ignores expiry is a glaring hole.

## Spec delta

- **`lexical-grammar.md` / `syntactic-grammar.md`.** `Bearer` becomes an admitted
  scheme. `scheme` gains a **parameterised form** `Bearer(secret = <string>)`
  (the `actor_decl` / `scheme` productions); `None`/`Internal` stay bare.
- **`static-semantics.md`.** Bearer admissibility is **HTTP-only**; a Bearer
  actor's `identity` type **must be string-constructible**
  (`karn.actor.bearer_identity_not_string_constructible`); the
  `Bearer(secret=…)` name must be present (`karn.actor.bearer_missing_secret`).
  The verify-then-mint rule: the seam mints the identity by constructing the
  declared type from the `sub` claim, fail-closed → 401.
- **`emission.md`.** The Bearer scheme-descriptor entry: extract `Authorization:
  Bearer`, HS256-verify the JWT against the sourced secret, validate `exp`/`nbf`,
  construct the identity type from `sub`, bind it for the body; any failure →
  `HttpResult.Unauthorized` before the handler runs. The minted identity threads
  to `<binder>.identity` (resolving the v0.45 lowering note). A small runtime
  helper (`verifyJwtHs256`) may be added to `runtime.ts`.
- **`runtime-library.md`.** Note the JWT-verify helper (if added) and that the
  Bearer secret is sourced via `Secrets`; no new public types (reuse
  `Unauthorized`).
- **`diagnostics.md`.** New `karn.actor.bearer_*` codes
  (`bearer_identity_not_string_constructible`, `bearer_missing_secret`); reuse
  `scheme_not_admissible` for non-HTTP Bearer.
- **LSP / tooling.** The `Bearer(secret = …)` config participates in
  hover/semantic-tokens; the secret string is a plain literal. Scope per the
  cross-component drift guard.

## Tests

- **Positive:** a `from http` route `by u: User` with
  `actor User { auth = Bearer(secret = "AUTH_JWT_SECRET"), identity = UserId }`
  reading `u.identity`; golden emission of the verification seam (header extract,
  HS256 verify, `exp` check, identity construction, the 401 path) and the handler
  receiving the minted identity.
- **Negative:** Bearer on a cron handler (`scheme_not_admissible`); a Bearer
  actor with `identity = Int` (`bearer_identity_not_string_constructible`); a
  `Bearer` with no `secret` (`bearer_missing_secret`).
- **Golden / security:** prove **fail-closed** — no/invalid/expired token ⇒ 401
  and the handler body is *not* called; prove the signature is actually verified
  (a tampered token fails); prove the secret is read via `Secrets` and never
  appears in a response or log.

## Diagnostics / Risks

- **Crypto correctness.** HS256 via WebCrypto must use `crypto.subtle.verify`
  (constant-time) and reject `alg: none` / algorithm confusion. Mitigation: a
  fixed, compiler-emitted verifier (no app input to the algorithm); golden +
  security review pin it.
- **Boundary capability access.** Sourcing the secret before `compose` is a new
  emission shape. Mitigation: instantiate `SecretsProvider(env)` exactly as
  compose does; bundle/worker parity covered by golden on both targets.
- **Identity-from-claim soundness.** The minted value must construct through the
  identity type's refinement (fail-closed on violation), not bypass it.
  Mitigation: reuse the path-param `.of` machinery; negative fixture for a
  refinement-violating `sub`.

## Security / threat model

*Gates the slice (ADR 0076).*

- **Guarantees.** A handler body runs only with a verified, sealed identity: the
  JWT signature is HS256-verified against the sourced secret, `exp`/`nbf` are
  enforced, and the `sub` is validated through the identity type. Failure is
  fail-closed → 401; the raw token never reaches the body (only the minted
  identity does).
- **Secret handling.** The signing secret is sourced via `Secrets` (ADR 0021),
  lives only in the seam, and is never logged or returned.
- **Explicitly out.** RS256/asymmetric + JWKS rotation, opaque-token lookup,
  refresh tokens, audience/issuer pinning beyond `exp`/`nbf`, rate limiting, and
  the 403 authorisation-invariant split (Q3).

## New decision records this increment adds

- **ADR — BearerToken is compiler-generated JWT/HS256; identity is a claim
  through the identity type; HTTP-only.** Admits `Bearer` to the scheme set with
  a JWT/HS256 verifier, the `sub` claim minted through a string-constructible
  identity, the secret named on `Bearer(secret=…)` and sourced via `Secrets`,
  fail-closed → 401.

## Definition of done

`actor … { auth = Bearer(secret = "…"), identity = T }` parses, resolves, and
checks (Bearer admitted, HTTP-only, identity string-constructible, secret
required); a `by u: User` HTTP handler emits the fail-closed JWT/HS256
verification seam — header extract, signature + `exp`/`nbf` verification, identity
construction from `sub`, 401 on any failure — with golden emission proving the
body is not reached without a verified identity; `<binder>.identity` lowers to the
real minted value (resolving the v0.45 note); the secret is sourced via `Secrets`
and never leaked; the `karn.actor.bearer_*` diagnostics land; the slice ADR lands
and `design/tracks/actors.md` marks slice 2 done; LSP covers the new surface;
spec chapters, guides, and changelog are updated; `cargo build`/`test`/clippy +
`cargo fmt --check` and CI stay green; `scripts/bump-version.sh` sets the version;
`/security-review` + `/code-review` gate the verification-bearing changes
