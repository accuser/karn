# v0.103 ‚Äî the `from WebSocket` protocol: edge-authenticated upgrade, held-connection transfer to an agent, and the `TestConnection` runtime (real-time track, slice 3)

- **Scope:** a single increment ‚Äî the `service <Name> from WebSocket(in:, out:)`
  protocol: the HTTP upgrade authenticated at the edge via `by`, the `on open`
  handler receiving a fresh `Connection[out]` and transferring it to an agent,
  inbound frames arriving as ordinary typed agent messages, the bundle/test
  `TestConnection` (capture-and-inspect), and the Workers hibernatable-WebSocket
  mapping. **Language + checker + emitter + runtime, both targets.**
- **Posture: the track's safety boundary.** A WebSocket upgrade authenticates at
  the service edge **before** a client holds a live channel into an agent ‚Äî a
  wrong shape is a vulnerability (the actor-track trigger). This slice **carries
  the threat model (¬ß5) and runs `/security-review` + `/code-review`** before it
  lands, per [ADR 0076](../decisions/0076-feature-track-posture.md)'s
  security-bearing-slice rule.
- **Realises:** [`design/tracks/websocket.md`](../tracks/websocket.md) ¬ß3.6‚Äì¬ß3.7
  and **Slice 3** of its ¬ß6 decomposition. Settles the track's **Q5** (held-value
  fault paths ‚Äî the *cross-context* remainder slice 2 deferred), **Q7**
  (hibernation re-acquisition), and **Q8** (the upgrade's auth carrier).
- **Builds on:** [ADR 0130](../decisions/0130-held-resource-linearity.md) (the
  `Connection[F]` type + linearity pass ‚Äî slice 2; the `on open` handler receives a
  held `Connection` as an owned param, the discipline governs it from there); [ADR
  0077](../decisions/0077-service-protocol-on-header.md) (`service ‚Ä¶ from
  <protocol>`, moved to the header *specifically* so it generalises to WebSocket);
  the **actor track** (ADRs 0080‚Äì0092 ‚Äî the `by` clause and the Bearer/Signature/
  None/Internal auth schemes the edge reuses).
- **Authorises its own ADR:** the `from WebSocket` protocol surface, edge-auth-
  before-accept, the auth carrier, connection-transfer, and the hibernation
  mapping. **Number assigned at PR-cut** (provisionally 0131), per the
  no-reservation rule.

## 1. What this is ‚Äî the live channel, fail-closed

Slices 0‚Äì2 built the primitives with no socket. This slice is the socket: a
service whose protocol is `WebSocket(in:, out:)` performs the HTTP upgrade,
**authenticates the actor at the edge before accepting the connection**, and
hands a fresh `Connection[out]` to an `on open` handler, which transfers it into
an agent. Thereafter inbound frames "arrive at the agent as ordinary typed
messages, indistinguishable in shape from messages invoked by other agents or by
tests" (¬ß7) ‚Äî so a frame handler is unit-testable in isolation, and the held
connection lives in agent state under the linearity discipline slice 2 built.

```bynk
service ChatGateway from WebSocket(in: ClientFrame, out: ServerFrame) {
  on open by user: Participant (params: { roomId: RoomId })
      given Rooms: Ref[Room] {
    <- Rooms(params.roomId).join(user.identity, connection)
  }
}

agent Room {
  key id: RoomId
  store members:     Set[UserId]
  store connections: Map[UserId, Connection[ServerFrame]]

  on join(u: UserId, conn: Connection[ServerFrame]) given Events {
    members.add(u)
    connections.put(u, conn)          -- conn: owned ‚Üí consumed (stored)
    Events.emit(UserJoined { ... })
  }
  on leave(u: UserId, _conn: Connection[ServerFrame]) { connections.remove(u) ... }
}
```

## 2. Scope and non-goals

**In scope.**

- The **`WebSocket(in:, out:)` protocol** ‚Äî a `ServiceProtocol::WebSocket { in_type,
  out_type }` on the service header (`bynk-syntax/src/ast.rs:488`, beside
  `Queue { name }`), binding the inbound frame type `in:` and the server‚Üíclient
  frame type `out:`.
- The **`on open` handler** ‚Äî a new `HandlerKind::Open` (`ast.rs:745`), admitted
  only on a `WebSocket` service, carrying a mandatory `by` clause (edge auth) and
  receiving an **owned `Connection[out]`** binding (the held-param exemption slice 2
  added, `validate.rs:2862`). It must dispose of the connection ‚Äî the canonical
  disposal is transfer into an agent (`Rooms(id).join(‚Ä¶, connection)`), which the
  linearity pass already enforces.
- **Inbound frames as typed agent messages** ‚Äî a frame decodes to the `in:` type
  and dispatches to the matching agent handler (`on join`/`on post`/`on leave`);
  the routing is the ¬ß7 "frames are messages" model, reusing the boundary-decode
  discipline against the declared `in:`.
- **`connection.send(frame)` / `connection.close()`** ‚Äî already typed and lowered
  (slice 2); this slice gives them a live transport.
- **`TestConnection`** ‚Äî the bundle/test `Connection<F>` implementation: a
  capture-and-inspect channel recording what was `send`-/`close`-d, so a streamed
  conversation is assertable (design notes ¬ß20).
- **The Workers hibernatable-WebSocket mapping** ‚Äî the upgrade in the stateless
  Worker, the server socket accepted into the addressed Durable Object (the agent)
  via the hibernatable API, and the stored connection re-associated on wake (Q7).
- **Edge auth carrier (Q8)** ‚Äî which request element a WS upgrade reads the actor
  credential from, given a browser cannot set an `Authorization` header.
- The ¬ß20 chat-room **type-checking** end-to-end as the protocol's proof (the full
  *runtime* closure, incl. broadcast-to-all, is slice 4 ‚Äî see non-goals).
- Spec-in-place for ¬ß7's WebSocket protocol; a `from WebSocket` reference; an ADR;
  the security review.

**Non-goals (and why).**

- **The broadcast-to-all-connections runtime (`connections.values.parTraverse(c =>
  c.send(‚Ä¶))`)** ‚Äî the `on post` fan-out needs the **held-aware iteration borrow
  surface** (`forEach`/`parTraverse` lending `&Connection` refs) that slice 2
  explicitly deferred (ADR 0130). The linearity pass already carries the borrow
  machinery; wiring storage iteration to lend held borrows is the remaining piece,
  and it lands with the **slice-4 closure** where the chat-room runs end-to-end.
  Slice 3 proves single-connection send/close over a live transport and the ¬ß20
  type-check; slice 4 makes it broadcast and run.
- **The Events/Alarm protocols** ‚Äî sibling future tracks (ADR 0076); out here.
- **A browser-side WebSocket *client*** ‚Äî `Connection` is server-held; client code
  is out of charter (¬ß19).
- **New held types** ‚Äî `Connection` stays the one instance of the closed kind.
- **Generalising the auth carrier beyond what WS needs** ‚Äî the carrier decision
  (D-Q8) is scoped to the WS boundary; it does not change HTTP auth.

## 3. Internal architecture

### 3.1 The protocol surface and handler model

`ServiceProtocol::WebSocket { in_type: TypeRef, out_type: TypeRef }` carries the
frame types on the header (ADR 0077's generalisation point). Two handler shapes:

- **`on open by <actor> (params) given ‚Ä¶`** ‚Äî `HandlerKind::Open`, exactly one per
  service, **`by` mandatory** (no anonymous upgrade by default; ¬ß5). It receives an
  implicit **`connection: Connection[out]`** owned binding and `params` decoded
  from the upgrade request (query/path), and must dispose of `connection` (transfer
  to an agent, the ¬ß2.9 discipline).
- **inbound frames** are *not* service handlers ‚Äî after transfer, the **agent**
  that owns the connection receives them as its own message handlers, the frame
  decoded to a variant of `in:` and dispatched. This keeps frame handlers
  indistinguishable from agent-to-agent messages and directly unit-testable.

### 3.2 Edge auth, fail-closed (the security crux)

The upgrade runs the **`by` auth seam first** ‚Äî the same Bearer/Signature
verification HTTP routes use (`actors.rs:157` `bearer_seam_for` /
`signature_seam_for`), extracted at emit time (`workers_entry.rs:54`). **Only on
success is the socket accepted**; on failure the handshake is rejected (the `101`
is never sent, no `Connection` is minted, `on open` never runs). The protocol
shape makes "accept-then-authenticate" **unrepresentable**: there is no seam
between accept and the `by` check.

### 3.3 Platform mapping (Workers) and TestConnection (bundle)

- **Workers:** the upgrade happens in the stateless Worker ‚Äî `new
  WebSocketPair()`, the `by` seam, then `state.acceptWebSocket(server)` into the
  addressed Durable Object (the agent), per Cloudflare's hibernatable API. Inbound
  `webSocketMessage(ws, msg)` decodes to `in:` and dispatches to the agent handler;
  the stored `Connection` is the server socket. The upgrade lives beside the
  existing `fetch` route dispatch (`workers_entry.rs`).
- **Bundle/test:** no DO, no upgrade. `TestConnection` implements `Connection<F>`
  (`connection.ts`), recording `send`/`close` into an inspectable buffer; a test
  harness drives `on open` with a `TestConnection` and feeds inbound frames to the
  agent ‚Äî "the runtime-managed lifecycle in production becomes a capture-and-inspect
  mechanism in tests" (¬ß20).

## 4. Decisions

**[DECISION A ‚Äî the security crux] Edge-auth-before-accept is structural, and `by`
is mandatory on `on open`.** A `from WebSocket` service with an `on open` handler
lacking a `by` clause is a compile error (mirroring `bynk.actor.missing_by_on_http`).
The emitted upgrade runs the auth seam and only `acceptWebSocket`s on success.
*Recommended;* this is the slice's reason for `/security-review`, and the one
shape a vulnerability would hide in.

**[DECISION B ‚Äî Q8, the auth carrier] A WS upgrade reads its credential from a
WS-native carrier, not the `Authorization` header; the scheme declares it.** A
browser `WebSocket` cannot set request headers, so a `Bearer` actor's
`Authorization: Bearer ‚Ä¶` has no carrier on the handshake. *Recommendation:* at the
`from WebSocket` boundary, read the Bearer token from the **`Sec-WebSocket-Protocol`
subprotocol** (the one header a browser *can* set, via `new WebSocket(url,
[protocol])`), echoing the selected subprotocol back on accept; **reject** a
silent `Authorization`-over-WS assumption, and **reject** a `Signature` scheme at
the WS boundary (HMAC-over-body has no handshake body). A query-parameter token is
admitted only behind an explicit opt-in (query strings land in logs). The actor
scheme gains a WS-carrier facet so the choice is declared, not implicit. *This is
the call I most want settled in `/security-review`.*

**[DECISION C ‚Äî Q7, hibernation re-acquisition] Keep it transparent in the
language; absorb re-association in the platform binding.** ¬ß2.9.6 promises a stored
`Connection` is restored transparently, but Cloudflare re-presents sockets via
`getWebSockets()` on wake, not as a live JS reference. *Recommendation:* the
language keeps "stored value in, stored value out"; the **platform binding** keys
the re-association ‚Äî each `Connection` carries a stored id via
`serializeAttachment`, and the binding maps the stored id to the re-presented
socket on wake, entirely behind the `Connection<F>` interface. The platform-mapping
slice proves transparency with an end-to-end hibernate/wake test. If the stored-id
indirection cannot be fully hidden, that is the one place "transparent" leaks ‚Äî flagged.

**[DECISION D ‚Äî Q5, the cross-context fault remainder] A connection mid-transfer is
owned by exactly one side at the fault instant; the owner's runtime closes it.**
Slice 2 settled the within-handler rules; the transfer from the service edge into
the agent DO is a **platform handoff** (`acceptWebSocket`), not a same-address-space
move. *Recommendation:* before the handoff the **edge** owns the socket (and closes
it if the upgrade/auth faults); after, the **DO** owns it (and the agent's stored
state rolls back transactionally on a handler fault, ADR 0109). The single-owner
invariant rests on the handoff's atomicity ‚Äî the language's move semantics describe
the source side, the platform binding guarantees the destination. A
`Sagas.compensate` on a connection already in agent state is admitted; on one still
held at the edge, statically rejected (slice 2).

**[DECISION E ‚Äî inbound frame routing] A frame decodes to the `in:` sum and
dispatches to the agent handler whose name matches the variant.** `in: ClientFrame
= Join | Post | Leave` routes a decoded `Post` to the agent's `on post`. Decoding
validates against `in:` (the boundary-decode discipline); an undecodable frame is
rejected, never reaching a handler. *Recommended;* it is the ¬ß7 "frames are typed
messages" model, and keeps the trust boundary at the decode.

## 5. Security and threat model (the `/security-review` brief)

- **Unauthenticated channel into an agent.** *Defence:* D-A ‚Äî auth-before-accept is
  structural and fail-closed; no frame reaches an agent before `by` verifies the
  actor. *Review focus:* that the emitted upgrade has no path from `WebSocketPair`
  to `acceptWebSocket` that skips the seam.
- **Auth carrier confusion (Q8).** A credential in the wrong place (a logged query
  string, a header the browser silently drops) is a silent auth bypass or leak.
  *Defence:* D-B ‚Äî a declared WS-native carrier; reject header-only schemes at the
  boundary. *Review focus:* the subprotocol echo, and that `Signature` is refused.
- **Connection-leak resource exhaustion.** A held connection is a real resource;
  the linearity pass (slice 2) rejects an undisposed one in normal flow, and D-D
  closes a handler-owned connection on abnormal exit. *Review focus:* the fault path
  closes the edge socket when auth/upgrade faults.
- **Frame-type confusion / trusting client bytes.** *Defence:* E ‚Äî inbound frames
  decode-and-validate against `in:` before dispatch. *Review focus:* the decode is
  total (a malformed/oversized frame is rejected, not coerced).
- **Per-agent connection caps / unbounded sockets.** A platform concern named here
  so the slice does not silently omit it; the DO enforces a cap.

## 7. Tests

- **Positive (bundle, via `TestConnection`):** the ¬ß20 chat-room *type-checks*; a
  WS service authenticates an actor, `on open` transfers the connection to an agent,
  an inbound `join`/`post` frame dispatches to the agent handler, the agent
  `connection.send`s, and the `TestConnection` records the exact frames sent ‚Äî
  assertable in a test.
- **Negative:** `on open` without `by` (D-A); a `Signature` actor at the WS boundary
  (D-B); a `from WebSocket` service on a target that cannot host it (the platform-lock
  diagnostic, ADR 0017); an inbound frame that does not decode to `in:`.
- **Workers emission:** the upgrade + `acceptWebSocket` + hibernatable `webSocketMessage`
  dispatch emit and `tsc --strict`-check against the runtime `Connection` interface and
  the Workers types; a hibernate/wake re-association test (D-C).
- Full `cargo test --workspace` **with `BYNK_REQUIRE_TSC=1`** (the gate CI enforces;
  every emitted fixture `tsc --strict`-verified) + `fmt` + `clippy -D warnings`.

## 8. Risks & mitigations

- **Security shape of the upgrade.** The one place a wrong protocol shape is a
  vulnerability. *Mitigation:* D-A makes auth-before-accept structural; slice gated
  by `/security-review`; threat model stated (¬ß5).
- **The auth carrier is genuinely unsettled (Q8).** *Mitigation:* D-B proposes the
  subprotocol carrier and refuses header-only schemes; it is the explicit review focus.
- **Hibernation transparency could leak (Q7).** *Mitigation:* D-C confines
  re-association to the platform binding behind the `Connection` interface; an
  end-to-end hibernate/wake test proves it.
- **Slice size.** The protocol + auth + upgrade + hibernation + `TestConnection` is
  large. *Mitigation:* broadcast-iteration is explicitly deferred to slice 4; this
  slice proves single-connection transport + the ¬ß20 type-check, not the full
  runtime fan-out.
- **`TestConnection`/Workers parity.** Two `Connection` implementations to keep in
  step. *Mitigation:* both implement the one `connection.ts` interface; a conformance
  shape shared by bundle and workers fixtures.
