# v0.50 — The `by` binder is optional: `by <Actor>` for anonymous / identity-not-consumed handlers

- **Status:** Draft (proposal). **Merging is approval to build.** *(A small surface refinement to
  the actors `by` clause — amends ADR 0082. Standalone increment, not a feature-track slice. Touches
  parser/AST/checker/formatter + the grammar; no new emission and no emitted-output change for
  existing handlers.)*
- **Sequences after the actors slices in flight** (v0.45 Foundations, v0.47 BearerToken). Independent
  of the unbuilt slices (Signature, multi-actor); they inherit the optional binder for free.

## Context — the binder is dead weight on anonymous routes

Foundations made HTTP require an explicit actor — a public route writes `by v: Visitor`. But
`Visitor` is the `None` scheme, whose identity is `()`: there is nothing to read, so the binder `v`
is always unused. Every public route carries a dead name:

```karn
service api from http {
  on GET("/ping")        by v: Visitor () -> Effect[HttpResult[String]] { Ok("pong") }
  on GET("/notes/:id")   by v: Visitor (id: String) -> Effect[HttpResult[String]] { NotFound }
}
```

The dead `v` is pure ceremony, repeated on every anonymous endpoint. The same shows up beyond
`Visitor`: a route that *authenticates but doesn't need who* (a Bearer gate that only cares the
caller is valid) still has to bind an identity it never reads.

## The reframing — decline to *capture*, not to *declare*

ADR 0082 made the identity a **named binding, never ambient** — the anti-pattern was Yesod's
`requireAuthId` reaching into hidden monad state; the requirement must be visible in the signature.
A **binder-less** `by Visitor` does **not** reintroduce ambient authority: the actor and its scheme
are still declared explicitly at the boundary, plainly in the handler signature, and verified before
the body. You are only declining to *capture* an identity value you don't use — not hiding the
contract. So the optional binder is fully compatible with 0082's intent; it removes ceremony, not
visibility.

## Builds on (verified)

- **The v0.45 `by` clause** (ADR 0082) — `by <name>: <Actor>` after the protocol config; HTTP
  requires `by`. This proposal makes the `<name>:` part optional; everything else (HTTP-requires-`by`,
  per-protocol defaults, two-phase verify) is unchanged.
- **The v0.47 Bearer seam** keys off the **actor**, not the binder (`bearer_seam_for` reads the
  `by` clause's actor; the binder only matters for `<binder>.identity` lowering). So verification
  still happens with no binder — the seam needs no change beyond not minting a named binding.
- **The `_` unused-slot convention** — the language already uses `_` for params/lets/match arms that
  aren't consumed; the optional binder is the same idea applied to the actor binding (see the decision
  on why we make the binder optional rather than bless `by _: Actor`).

## Decisions (open — settle at review)

**[DECISION] Optional binder vs an `_` binder vs status quo.** *Rec:* **make the binder optional** —
`by <Actor>` (no identity captured) and `by <name>: <Actor>` (captured). *Alternatives:* (a) reuse the
unused marker, `by _: <Actor>` — consistent with `_` elsewhere, but once the binder is optional it
leaves *two* spellings for "anonymous" (`by Visitor` and `by _: Visitor`) that the formatter must
reconcile; (b) status quo (always require a binder) — the ceremony this fixes. The recommendation
gives exactly one spelling per intent: omit the binder when you don't consume the identity, name it
when you do. `_` is **not** admitted as a binder (use the binder-less form).

**[DECISION] Does the binder-less form apply to all schemes, or only `None`?** *Rec:* **all schemes,
uniformly.** `by User` (Bearer, binder-less) is a legitimate "verify the token, don't need the
identity" gate — verification still runs fail-closed; the minted identity is simply discarded.
Restricting the binder-less form to `None` would be a special case with no payoff. *Alternative:*
`None`-only — narrower, but arbitrary.

**[DECISION] Migrate existing `by v: Visitor` usages?** *Rec:* **adopt the terser `by Visitor` in the
docs and worked examples** (the readability win that prompted this), and **leave fixtures as the
author wrote them** — both forms are valid, so there is no forced migration; keep a few fixtures on
each form to pin both. *Alternative:* mass-migrate every usage — churn for no behavioural gain.

## Spec delta

- **`syntactic-grammar.md` / tree-sitter.** `by_clause` becomes `"by" (identifier ":")? <actor>` —
  the binder + `:` are optional. One-token lookahead disambiguates (`:` after the first identifier ⇒
  it is the binder; otherwise that identifier is the actor).
- **`static-semantics.md`.** The `by` clause may omit its binder; when omitted, no identity binding
  enters the handler scope (and `karn.actor.binder_shadows_param` cannot arise). HTTP still requires
  a `by` clause (`karn.actor.missing_by_on_http` unchanged) — only the binder is optional. Per-protocol
  default-actor inheritance (omitting `by` entirely on non-HTTP) is unchanged.
- **AST.** `ByClause.binder: Option<Ident>`.
- **`emission.md`.** No change: a binder-less handler mints no named identity (nothing to thread);
  the verification seam is unchanged (it keys off the actor). Emitted output for existing
  `by <name>: <Actor>` handlers is byte-identical.
- **LSP / formatter.** The formatter emits `by <Actor>` when the binder is absent, `by <name>: <Actor>`
  when present; the binding index records the binder only when present.

## Tests

- **Positive:** a binder-less `by Visitor` HTTP route (golden: emits identically to the named form
  today, since `Visitor` mints nothing); a binder-less `by` on a non-HTTP handler; a binder-less
  Bearer `by User` (verifies fail-closed, discards the identity — golden shows the seam runs, no
  identity threaded). Keep at least one named `by v: Visitor` fixture so both spellings stay pinned.
- **Negative:** HTTP handler with no `by` at all still errors (`missing_by_on_http`); `by _: Visitor`
  is rejected (the binder, if written, must be a real name — guides authors to the binder-less form).
- **Formatter:** round-trip both forms; `by Visitor` stays binder-less, `by v: Visitor` keeps its name.

## Diagnostics / Risks

- **Parse ambiguity.** None in practice — the `:`-lookahead is unambiguous, and the actor is always
  followed by the parameter `(`. *Mitigation:* a corpus test for both forms.
- **A second way to say one thing.** Mitigated by *not* admitting `by _: Actor` (one spelling per
  intent) and by the formatter normalising presentation.
- **Surface-settled drift.** This amends a settled ADR; recorded as a new decision record that
  references 0082 rather than editing it (records are immutable).

## New decision records this increment adds

- **One ADR — the `by` binder is optional** (amends ADR 0082). The binder/`:` may be omitted when the
  handler does not consume the identity; the contract stays explicit at the boundary, so this is a
  ceremony reduction, not a return to ambient authority. `_` is not admitted as a binder. Applies to
  all schemes.

## Out of scope (→ unchanged / later)

- **HTTP-requires-`by`** and per-protocol default actors (omitting `by` entirely) — unchanged.
- **The unbuilt actors slices** (Signature, multi-actor, invariants) — they inherit the optional
  binder; no coupling.

## Definition of done

`by <Actor>` (binder-less) parses, resolves, and checks across all four handler forms and all admitted
schemes; `by <name>: <Actor>` is unchanged; `by _: <Actor>` is rejected with a note pointing at the
binder-less form; HTTP still requires a `by` clause; a binder-less handler mints no identity and emits
byte-identically to its named equivalent (golden); the formatter round-trips both forms; the
tree-sitter grammar + corpus, spec chapters, diagnostics, and LSP indices cover the optional binder;
the docs/worked examples adopt the terser `by Visitor`; the amending ADR lands; `cargo build`/`test`/
clippy + `cargo fmt --check` and CI stay green; `scripts/bump-version.sh` sets the version. The
implementing PR **deletes this file**.
