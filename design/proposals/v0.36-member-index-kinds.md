# v0.36 — Member index kinds (methods, record fields, capability ops)

- **Status:** Draft (proposal). **Merging is approval to build.** *(Tooling-queue item 3, the **members**
  slice. Index the three parent-scoped identifier kinds — instance methods, record fields, capability op
  names — as first-class symbols, so references, rename, semantic tokens, and the v0.34 call-hierarchy
  method edges all extend to them. Locals and generic type params are the queue's other slice and stay
  deferred — see Out of scope.)*

## Context

The binding index records only top-level kinds (`Type, Fn, Capability, Service, Agent, Provider`).
Instance methods (`x.m()`), record fields (`r.field`), and capability ops (`Cap.op()`) are deferred
across v0.25/v0.27/v0.28/v0.34: a method registers as an attribution-only `add_owner` (`index.rs`),
never an `add_def`; fields and ops record nothing. So go-to-def, references, rename, and semantic tokens
all go dark on them, and call hierarchy skips method calls (`CallEdge` takes `Fn` callees only).

All three are **parent-scoped** (`T.m`, `T.field`, `Cap.op`) and share one enabling fact: the parent's
type is **already resolved at the use site in the checker** — `check_method_call` has `recv_ty`
(`calls.rs:1085`, name at `:1117`), `check_field_access` has the record type (`expressions.rs:1554`,
type at `:1627`), op dispatch has the capability (`calls.rs:677`). So the use site can record a ref
**already spelled `"Parent.member"`**, and the def walk can register the matching key — both flat keys in
the existing `SymbolKey { unit, kind, name }`, resolved by the **existing qualification machinery**, no
new key shape and no assembly-time parent injection.

## Builds on (verified — scout)

- **Spans are all in hand.** Method def `method_name.span` (`ast.rs` `FnName::Method`), call `method.span`
  (`calls.rs:932`); field def `RecordField.name.span` (`ast.rs:690`), access `field.span`
  (`expressions.rs:1554`); op def `CapabilityOp.name.span`, call `method.span` (`calls.rs:692`).
- **The parent type is resolved at every use site** (the three checker sites above) — enough to form the
  `"Parent.member"` name. The `RefSink` (`ctx.refs`) is already threaded through these checker paths (it
  records type/capability refs there today), so recording is an added call, not new plumbing.
- **Qualification already walks `uses`/`consumes`.** `IndexBuilder::qualify_with` (`index.rs`) resolves a
  bare name against the current unit's merged tables (local → `uses` → `consumes`). A member def
  registered under its parent's unit with the compound name resolves from any unit that imports the
  parent — the *same* path top-level cross-file refs take. (Cross-context ops may need the
  `record_in_unit`/`flattened_caps` treatment capability refs already use — a [DECISION] below.)
- **Methods already have owner attribution.** `add_owner(unit, "T.m", path)` exists; adding `add_def`
  for the method populates `owner_keys` too, so methods become valid **callers and callees** in the
  v0.34 call graph for free.
- **Downstream reads are key-agnostic.** `references`, `rename` (`plan_rename`), `documentHighlight`, and
  `semantic_tokens` all key on `SymbolKey` — they extend the moment members are indexed.  `prepare_rename`
  explicitly *denies* these kinds today (`index_queries.rs`); the denial is lifted.
- **Locals are a separate, mostly-done system.** v0.31 `locals.rs` already serves goto-def, highlights,
  and tokens for locals; only rename is missing — independent of this increment.

## Decisions (open — settle at review)

**[DECISION] Key representation.** *Rec:* keep the flat `SymbolKey`; add three `SymbolKind` variants
**`Method`, `Field`, `CapabilityOp`**, each with a **compound `name = "Parent.member"`**. Distinct
variants (not one `Member`) let semantic tokens colour them (`Method`→`METHOD`, `Field`→`PROPERTY`,
`Op`→`METHOD`) and keep the index legible. No change to `qualify`/`SymbolKey` shape — the compound name
rides the existing machinery.

**[DECISION] Where refs are recorded.** *Rec:* **in the checker**, at the three dispatch sites, after the
parent type is resolved — `ctx.refs.record(member_span, kind, "T.m")` (bare; assembly qualifies). The
def is registered at the **project walk** (`symbols.rs`) alongside the existing top-level walk: methods
gain an `add_def` next to their `add_owner`; record bodies and capability ops are walked for their member
defs.

**[DECISION] Cross-context ops.** *Rec:* a `B.Cap.op` call resolves the op against the consumed unit; if
qualification of the bare compound name doesn't reach it, record the op ref `in_unit` (the providing
unit), mirroring `record_capability_clause_ref`'s `flattened_caps` handling. Verify against a
cross-context fixture; fall back to local-only ops if it proves fiddly (and note the gap).

**[DECISION] Call-hierarchy method edges.** *Rec:* extend the `CallEdge` build to accept **`Method`**
callees (and methods as callers, now that they're `add_def`'d). Op/dispatch callees follow the same
rule. This closes the v0.34 deferral in the same increment.

**[DECISION] Rename safety.** *Rec:* members join the existing rename validators unchanged (re-analyse +
index-equality-modulo-rename) — a compound-name symbol renames its bare member segment across all
qualified sites. Confirm the edit targets only the member segment of `"T.m"`, never the `T.`.

**[DECISION] Slicing.** *Rec:* land as **two PRs** under one design — **slice 1: methods** (+ their
call-hierarchy edges; the highest-value kind and the one that exercises the compound-key path end to
end), **slice 2: record fields + capability ops** (the same machinery, applied to the two access forms).
Each slice ships its own tests, changelog, and version bump; the proposal is deleted on slice 2.

**[DECISION] New decision record.** *Rec:* **ADR 0069** — member index kinds via compound names + three
`SymbolKind` variants, refs recorded at the checker's resolved use sites, resolved by the existing
qualification; call-hierarchy method edges; locals/generics deferred.

## Spec delta (sketch)

- **`design/karn-lsp-spec.md`** — extend the index/§3 coverage notes: methods, record fields, and
  capability ops are now index symbols (compound `Parent.member` keys), so references/rename/semantic
  tokens and call-hierarchy method edges cover them; locals (rename) and generic params remain deferred.

## Tests

- **`karnc` index** (`reference_index.rs` matrix): method def + call same-file and cross-file (via `uses`);
  field def + access, incl. on an imported type; op def + call in a context and in a provider; **shadowing**
  — two types with a same-named method index as distinct `"T1.m"`/`"T2.m"` keys. Each asserts def + all
  ref sites with name-segment spans.
- **`karnc` call-hierarchy** (`call_hierarchy_index.rs`): a method call now records a `CallEdge`; a method
  is a valid caller and callee.
- **`karn-lsp`**: `prepare_rename` accepts a member; `plan_rename` edits only the member segment across
  sites; `semantic_tokens` emits the new kinds. (Position/transport stays thin.)

## Diagnostics / Risks

- **Compound-name qualification** is the crux — but it reuses the proven top-level `uses`/`consumes` walk;
  the shadowing and cross-file fixtures are the guard. Cross-context ops are the one corner that may need
  `in_unit` recording (decision above).
- **Rename hitting the `T.` segment** — the edit must target the member segment only; covered by a test.
- **No new ambiguity class** — `"T.m"` is ambiguous only where `type T` itself already is.

## Out of scope (→ later)

- **Local-binding rename** — locals already serve goto/highlight/tokens via the v0.31 side-channel; only
  rename is missing, and it's a *scoped* validator change in the locals system, not the index. A clean
  separate increment.
- **Generic type parameters** — `TypeRef` spans are coarse (no per-name segment), so use-site references
  need an AST change; low LSP value (always in-scope, no cross-file refs). Deferred.
- Inlay hints gain nothing here (they key on inferred types at bindings, not on member defs/refs).

## Definition of done

Instance methods, record fields, and capability ops are first-class index symbols (compound
`Parent.member` keys, three new `SymbolKind` variants): go-to-definition, references, rename, and
semantic tokens cover them, and call hierarchy records method (and op) edges. Built on refs recorded at
the checker's resolved use sites and resolved by the existing qualification — no new key shape. `karnc`
index/call-hierarchy tests cover same-file, cross-file, shadowing, and provider/op cases; `karn-lsp`
tests cover member rename and tokens; `design/karn-lsp-spec.md` updated; ADR 0069 added; CI green;
changelog; version → 0.36.0. Lands as two PRs (methods; then fields + ops); the final PR **deletes this
file**.
