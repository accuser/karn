# v0.71 ‚Äî workerd debugging: `bynk dev --inspect` (debugging track, slice 3)

- **Track:** [debugging](../tracks/debugging.md), slice 3 ‚Äî the headline UX (debug the
  worker you actually deploy), and the slice on the track's *least-certain dependency*
  (open question 1, the `wrangler` inspector). **The spike already settled it** ‚Äî this
  slice is thin.
- **Realises ADR:** [0104](../decisions/0104-debug-launch-model.md) D3 (workerd target);
  **consumes** [0103](../decisions/0103-source-map-contract.md) and the v0.70
  per-statement handler maps.
- **Posture:** tooling. Adds an `--inspect` mode to `bynk dev`; no language surface.

## Why this slice is small

The track flagged two uncertainties for workerd debugging; the slice-3 **spike closed
both** against a real `wrangler dev` worker:

- **Open question 1 ‚Äî the inspector port.** `wrangler dev --inspector-port <N>` exposes
  a Chrome-DevTools-Protocol endpoint: `http://127.0.0.1:<N>/json` returns a target with
  `webSocketDebuggerUrl: ws://127.0.0.1:<N>/ws`. **Attach directly** ‚Äî no `bynk dev`-owned
  proxy. One gotcha: wrangler's inspector **requires an `Origin` header** on the WebSocket
  (`400 Bad Request: Expected 'Origin' header` without it). VS Code's JS debugger sends
  one; a hand-rolled client must too.
- **Open question 3 ‚Äî map composition through the bundle.** `wrangler`/esbuild **composes**
  the emitted `handlers.ts.map` into the worker bundle: the bundle's `index.js.map`
  `sources` include the `.bynk` file. A breakpoint mapped from `.bynk` binds and pauses on
  a real request. **No bynk-side bundling work** ‚Äî the toolchain does it.

And v0.70 made handler bodies map **per-statement**, so the breakpoint lands on the
handler *statement*, not the `service` line. The mechanism is fully proven; what remains
is a thin launch surface plus a regression guard.

## Current state

`bynk dev` already does almost everything (ADR 0096): it compiles in Workers mode to
`.bynk/dev/`, **writes the worker output through the map-aware writer** (so
`handlers.ts.map` lands on disk), selects the context's worker, `cd`s into its dir, and
runs `wrangler dev` (`bynk/src/dev.rs` ~157‚Äì166) ‚Äî already forwarding `--` passthrough
args. So `bynk dev -- --inspector-port 9229` *works today*; the spike used exactly that.
What's missing is a first-class flag, attach guidance, and an automated proof.

## What this slice does

1. **`bynk dev --inspect`** ‚Äî a first-class flag that injects `--inspector-port <port>`
   (default 9229; `--inspect-port <N>` overrides) into the `wrangler dev` invocation, and
   prints **attach guidance** before handing over: the inspector URL/port, the CDP `/json`
   discovery endpoint, and the `Origin`-header requirement. Mirrors v0.69's `bynkc test
   --inspect`. The existing `--` passthrough still works for power users.
2. **A headless CDP proof** ‚Äî productises the spike: build a one-context HTTP project,
   start `wrangler dev --inspector-port`, attach over CDP **with an `Origin` header**, set
   a breakpoint on a handler statement (resolved through the composed bundle map), send a
   request, and assert it **binds and pauses** at the matching `.bynk` line. Guarded on
   `wrangler` (+ workerd) availability ‚Äî skipped when absent so it never blocks CI (the
   v0.70 decode goldens are the *always-on* correctness guard for the maps themselves).
3. **`skipFiles` boundary ‚Äî surfaced, wired in slice 4.** Per ADR 0103 D5 the runtime and
   generated glue should be skip-stepped. That is a *launch-config* setting the attaching
   debugger owns; the CLI has no launch config, so slice 3 only *documents* the boundary
   (which files are toolchain machinery). Slice 4's `DebugConfigurationProvider` sets the
   actual `skipFiles`/`smartStep`.

## Decisions

**[DECISION A ‚Äî sign-off] A first-class `--inspect` flag, not just the `--` passthrough.**
Fork: (a) add `--inspect` (+ `--inspect-port`) to `bynk dev`; or (b) document `bynk dev --
--inspector-port`. **Recommend (a)** ‚Äî it gives a place to print the attach guidance (the
`Origin` gotcha especially), a sensible default port, and parity with `bynkc test
--inspect`; the passthrough stays for overrides. Cheap (a few clap lines + arg injection).

**[DECISION B ‚Äî sign-off] The CDP proof is availability-guarded, the decode goldens are
the CI floor.** The workerd attach needs `wrangler` + workerd (a download, possibly
network) that CI may not have, so the CDP test **skips when `wrangler` is unavailable** ‚Äî
like v0.69's Node-version guard. Correctness of the maps the attach relies on is already
guarded *always* by v0.70's in-process decode goldens (`handlers.ts.map` per-statement).
*Recommend as stated* ‚Äî the end-to-end attach is proven by the spike and the local/opt-in
test; CI never depends on a heavyweight external runtime.

**[DECISION C] `skipFiles` is slice 4.** The CLI surfaces the inspector; the debugger
client (the extension) owns the launch config. Slice 3 documents the boundary only.

**[DECISION D] Version ‚Äî recommend minor v0.71.** A new `bynk dev` mode, no language
change ‚Üí minor, over 0.70.0.

## Risks & mitigations

- **The `Origin`-header gotcha bites attach clients.** Anyone (or any tool) attaching a
  hand-rolled CDP client without `Origin` gets a 400. Mitigated by documenting it in the
  `--inspect` guidance and the docs, and by slice 4 relying on VS Code's JS debugger
  (which sends `Origin`).
- **wrangler/workerd absent or version-skewed in CI.** The CDP proof skips; the decode
  goldens keep the maps honest regardless. `bynk doctor` already gates `wrangler` for the
  deploy capability, so `bynk dev --inspect` inherits a clear "install wrangler" path.
- **Inspector-port collision.** A fixed default (9229) can clash. Mitigated by
  `--inspect-port` and by surfacing the chosen port in the guidance.
- **Map composition could regress** if wrangler/esbuild changes its bundler. Mitigated:
  the opt-in CDP test catches it locally; if it ever breaks, ADR 0103's open question 3
  fallback (emit a post-bundle map) is the documented escape.

## Spec & docs delta

- **Spec:** none (ADR 0104 governs the launch model).
- **Docs:** a `bynk dev --inspect` entry in the CLI reference (auto-generated from clap),
  and a short "Debugging a worker under `bynk dev`" note in the dev/tooling docs ‚Äî

  bundle map; note the `Origin`-header requirement for hand-rolled clients. Changelog row;
  currency banner with the bump. The one-click VS Code experience remains slice 4.
