# v0.29.1 — Refactor track, increment 1: characterization tests for the seams about to move

- **Status:** Draft (proposal). **Merging is approval to build.** *(First increment of the
  **refactor track** — the internal-quality paydown captured in
  `design/karn-refactor-proposal-queue.md`. This increment adds **no production code change at all**:
  it pins the pure helpers that the structural splits will relocate, so those splits are verifiable
  rather than merely plausible. It is the safety net for everything after it.)*

## Context — why this is first, and standalone

- The refactor queue's headline risk: `project.rs` / `checker.rs` / `emitter.rs` carry **~4 `#[test]`
  each**, and coverage is **almost entirely end-to-end** (golden fixtures over compiled output). The
  pure helpers the section-1 splits will move — `normalize_rel`, `unit_path_matches`,
  `canonicalise_cycle`, the consumes-cycle DFS, the duplicate TS emitter (`ts_type_ref_emit`,
  `escape_ts_string`) — are exercised **only transitively**. A behavioural change inside one during a
  move would not be localized; it would surface (if at all) as a fixture diff somewhere downstream.
- The fix is the standard refactoring discipline: **land characterization tests on `main` *ahead of*
  the move, green against today's code.** The move (v0.29.2+) then re-runs the *same* tests, still
  green — that before/after identity is the whole proof. Shipping the tests inside the move PR would
  forfeit the "before."
- This increment is **behaviour-preserving by construction** — it only adds `#[cfg(test)]` code. It
  follows the existing in-crate `super::`-importing pattern (`project.rs::platform_lock_tests`,
  `project.rs:7858`), so the helpers stay private; no visibility change to production items.

## Builds on (verified)

- **Existing seam tests are sparse and end-to-end.** `project.rs`'s only unit module is
  `platform_lock_tests` (`project.rs:7858`, 4 tests over `lock_violation`); the rest of the file's
  behaviour is covered through full-compile golden fixtures.
- **The helpers to pin are pure or near-pure** (verified signatures, `karnc/src/project.rs`):
  - `normalize_rel(p: &Path) -> PathBuf` (`:108`) — `.`/`..` collapse, separator normalization.
  - `unit_path_matches(rel_path: &Path, qualified_name: &str) -> bool` (`:3401`) — path ↔ qualified-name
    alignment (drives `check_path_name_alignment` / `check_test_path_alignment`).
  - `canonicalise_cycle(cycle: &[String]) -> Vec<String>` (`:4253`) — rotates a cycle to a canonical
    representative (so cycle diagnostics are deterministic regardless of discovery order).
  - `detect_consumes_cycles(&HashMap<String, Vec<String>>, &mut Vec<CompileError>)` (`:4184`) +
    `dfs_consumes` (`:4207`) — the dependency-cycle DFS; takes a plain adjacency map, so a synthetic
    graph fixture exercises it directly without a project on disk.
  - `commons_dir_for` (`:3250`), `ts_output_path` (`:3259`), `worker_dir_name` (`:3267`),
    `worker_handlers_source_path` / `worker_handlers_output_path` (`:3274` / `:3282`) — pure path maps.
  - `is_unpinned_range(range: &str) -> bool` (`:71`) — dependency version-pin check.
  - `ts_type_ref_emit(r: &TypeRef) -> String` (`:7626`), `ts_type_ref_emit_qualified` (`:7672`),
    `escape_ts_string(s: &str) -> String` (`:7611`), `sanitise_case_name` (`:7595`), `sanitise_suite`
    (`:6401`) — the **second TypeScript emitter** living in the test-emission helpers. Pinning these
    twice over (here, and against the real emitter's `escape_ts_string` at `emitter.rs:5561`) is also
    the **de-risking for item 7** (eliminate the duplicate emitter) — it makes the consolidation a
    test-anchored move instead of a leap.

## Settled

- **▸ Characterization, not specification.** These tests assert **what the code does today**, pinning
  current observable output — not what it *ought* to do. If a test encodes a latent bug, that is fine
  and intended: the move must preserve the bug; fixing it is a separate, visible change. (A discovered
  bug worth fixing is logged, not folded into a refactor.)
- **▸ Tests precede moves.** Each helper is pinned **before** the increment that relocates it. The
  pin's value is the before/after identity; co-shipping it with the move destroys that.

## Tests (this increment *is* the tests)

- **`project.rs` unit modules**, following the `platform_lock_tests` pattern (`#[cfg(test)] mod … {
  use super::*; }`):
  - `normalize_rel` — `.`/`..` collapse, trailing separators, already-normal paths, root-relative.
  - `unit_path_matches` — aligned path/name pairs (true), misaligned (false), the test-suffix and
    nested-directory cases that `check_*_path_alignment` rely on.
  - `canonicalise_cycle` — rotation to the canonical representative is **stable across input
    rotations** (the property that keeps cycle diagnostics deterministic); singleton and two-node
    cycles.
  - `detect_consumes_cycles` over synthetic adjacency maps — acyclic → no error; a 2-cycle and a
    3-cycle → exactly one canonicalized diagnostic each; **self-loop**; disjoint components.
  - path maps (`commons_dir_for`, `ts_output_path`, `worker_dir_name`, `worker_handlers_*_path`) —
    representative context names → expected paths; `is_unpinned_range` over pinned/unpinned/range
    inputs.
  - **the duplicate TS emitter** (`ts_type_ref_emit`, `ts_type_ref_emit_qualified`, `escape_ts_string`,
    `sanitise_case_name`, `sanitise_suite`) — base types, generic/`List`/`Map` refs, qualified refs,
    quote/backslash/newline/unicode escaping, identifier sanitization of awkward case names. **Mirror
    the escaping fixtures against `emitter.rs::escape_ts_string`** so the two emitters' current
    agreement (and any current *dis*agreement) is captured before item 7 merges them.
- **Coverage target:** every pure helper named in the section-1 split plan (`v0.29.2`) has at least one
  direct test before that split lands.
- **Not in scope:** integration/golden fixtures (already strong); `checker.rs` helper pins (their own
  later increment).

## Diagnostics / Risks

- **Over-specifying internals.** Pin **observable returns**, not incidental representation — e.g. for
  `canonicalise_cycle` assert the canonical-rotation *property*, not a brittle exact vector that an
  innocuous reformat would churn. Keep the tests robust to the move, since surviving the move unchanged
  is their entire job.
- **Pinning a latent bug.** If a helper does something subtly wrong, the test pins it as-is (by
  design). Note any such case in the test comment with a `// characterizes current behaviour; see …`
  marker so it is not mistaken for an endorsement, and file the fix separately.
- **Helper privacy.** All targets are private `fn`s reached via `super::*` — no `pub` widening, so the
  pins do not themselves enlarge the API surface the later splits then have to preserve.
