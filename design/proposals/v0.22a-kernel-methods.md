# v0.22a — Stdlib kernel methods (strings, `Option`/`Result`, numerics)

- **Status:** Draft (proposal). **Merging is approval to build.**
- **Builds on (verified):** **`Float` (v0.21, #29, landed)**; **the collection kernel +
  kernel-method machinery (0034/0037)** — the extension point this slice rides; v0.20a
  generics + v0.20b `List`/`Map`; **ADR 0041** (the value-methods rule); **ADR 0028**
  (open-narrow, *no bounds* — why a single generic `abs` is impossible).
- **Spec sections touched:** §4 (grammar — minor), §5 (static semantics), §7 (emission), §8
  (compilation model — `karn.string`), §9 (diagnostics), Appendix B (its own version-history
  entry — the v0.20a/b precedent).
- **Scope:** **purely additive surface — no boundary/wire change.** String ops + `concat`/
  `join`, `Option`/`Result` combinator methods, `Int`/`Float` value methods + `parse`
  statics. The typed `Json` codec, `JsonError`, type-app-on-statics, and the bare-`Int`
  tightening are **v0.22b**.
- **Why this is its own slice (0023 + the v0.20a/b precedent):** v0.22b carries a **mass
  re-bless** (the bare-`Int` tightening), and a re-bless is where regressions hide — keeping
  it out of this additive slice means v0.22a touches **no existing emission**. The two slices
  are independent; v0.22b's hardest item (`ConstructorCall` type-args) shares nothing with
  this one.

## Settled (recorded as ADRs ≥0045 — this slice: 0046, 0048)

- **▸ Strings stay built-in.** `String` is opaque (no char access in Karn), so its ops are
  compiler built-in methods lowering to TS string methods — the 0034/0037 hybrid posture,
  not Karn-written.
- **▸ Combinators & numeric helpers are value methods, not free functions** *(0041).* The
  only collision-free, ADR-consistent surface: free functions imported by bare name collide
  under `karn.resolve.duplicate_fn` (`resolver.rs:217`) — `karn.list` already exports
  `map`/`filter`/`find` (`firstparty.rs`), so `uses karn.list` + `uses karn.option` would
  fail to resolve. Built-in methods on the compiler-known receivers sidestep it **and chain
  on day one**. (0041's rule: operations on a value are methods; ways to obtain a value are
  statics.)

## Decisions (open — settle at review)

**[DECISION] `Option`/`Result` combinator surface.** *Rec:* **(a) built-in kernel methods**
on the `Option`/`Result` receivers — `o.map(f)`, `o.andThen(f)`, `o.getOrElse(x)`,
`o.isSome()` / `r.isOk()`, `r.mapErr(f)`, `o.okOr(e)` — extending the v0.20b kernel-method
machinery: zero collision surface, chaining day one, consistent with 0041. *Alternatives:*
(b) globally-disjoint free names (`mapOption`, …) — survives `duplicate_fn` but scales badly;
(c) module-qualified call syntax — new machinery. *Note:* (a) gives up the generics-
dogfooding the original v0.22 draft cited as "per 0037" — but 0037's actual reason (no
generic **user** methods) does not forbid **built-in** methods.

**[DECISION] `Int`/`Float` helper surface.** A single generic `abs[A](a: A) -> A` is
**impossible** — 0028 is open-narrow with **no bounds** (`karn.generics.no_bounds`), so a
generic body cannot compare `a` to zero. *Rec:* **built-in value methods** lowering to
`Math.*`, per 0041: `i.abs()` / `f.abs()`, `a.min(b)` / `a.max(b)`, `f.clamp(lo, hi)`, and
`f.isNaN()` / `f.isFinite()` (host functions, hence **value methods** — *not* `Float.`-
statics). `Int.parse(s) -> Option[Int]` / `Float.parse(s) -> Option[Float]` stay **statics**
(0041 names `Float.parse`). **Pin parse semantics:** full-string parse (reject `parseFloat`'s
trailing-garbage laxity), overflow → `None`.

**[DECISION] String surface + pinned semantics (each footgun pinned normatively).**
Built-in receiver methods, **UTF-16 code units** unless noted: `s.length()` *(called method —
kernel convention, `checker.rs:3537`; not the property `s.length`)*, `s.split(sep) ->
List[String]`, `s.trim()`, `s.contains(sub)`, `s.startsWith(sub)` / `s.endsWith(sub)`,
`s.replace(a, b)`, `s.slice(lo, hi)`, `s.indexOf(sub) -> Option[Int]`, `s.toUpper()` /
`s.toLower()`, `s.chars()`. The footguns:
- **No concatenation exists in Karn** — `+` is numeric-only (`check_binop`, Add grouped with
  Sub/Mul/Div). *Rec:* add `s.concat(t)` (additive, **no operator change**) and `join(parts:
  List[String], sep: String) -> String` — the latter a **Karn-written helper in
  `karn.string`** (over `concat` + `fold`), which justifies that commons' existence.
- **`s.replace(a, b)`** — TS string-pattern `replace` hits the **first** occurrence only.
  *Rec:* **replace-all** (pin it).
- **`s.chars()`** — *Rec:* **code points** (`[...s]`), not `split("")` code units. Normative
  note: `s.length() != s.chars().length()` for astral characters.
- **`s.slice(lo, hi)`** — TS `slice` wraps negative indices. *Rec:* **reject negatives**
  (sub-decision — or inherit the wrap; pin one).

## Spec delta (sketch — normative prose written in-place during implementation)

- **§4 Grammar** — minor (no new literal forms).
- **§5 Static semantics** — string-method typing (UTF-16 code units, normative); `Option`/
  `Result` **method** typing; numeric value-method + `parse`-static typing.
- **§7 Emission** — string ops → TS string methods; `Option`/`Result`/numeric methods →
  kernel lowerings (incl. `Math.*`); `join`/`concat`.
- **§8 Compilation model** — `karn.string` first-party commons (for `join`). *(No
  `karn.option`/`karn.result`/`karn.int`/`karn.float` commons — those surfaces are kernel
  methods.)*
- **§9 Diagnostics** — none new this slice (`parse` returns `None`, not a diagnostic).

## Diagnostics

None new. *(`karn.types.uninferable_type_arg` belongs to v0.22b's `decode[T]`.)*

## Fixtures (shape — not normative emission)

- **Positive** — string ops (`split`/`trim`/`slice`/`contains`/**`concat`**/`replace`-all/
  `chars` code-points/**`join`**); numeric methods (`abs`/`min`/`clamp`) + `Int.parse`/
  `Float.parse` round-trips; `Option`/`Result` **method** chains (`o.map(f).getOrElse(x)`).
  Each passes `tsc --strict`.
- **Additive guard** — **all** prior fixtures byte-stable (this slice changes no existing
  emission — purely new surface).

## Risks
- **String semantics** — UTF-16 code units, `replace`-all, `chars` code-points, `slice`
  negatives: pin each normatively or inherit a TS surprise.
- **Kernel-method resolution** — adding methods on `Option`/`Result`/numeric receivers must
  not perturb existing collection method resolution.

## New decision records this slice adds

- **0046** — strings built-in; UTF-16 code units; pinned `concat`/`join`/`replace`-all/
  `chars` code-points/`slice` semantics.
- **0048** — `Option`/`Result` combinators and `Int`/`Float` helpers are built-in **value
  methods** (per 0041), not stdlib commons free functions.
- *(Final numbering at implementation; numbers reflect the combined v0.22 plan, with the
  codec records reserved for v0.22b.)*

## Out of scope (→ v0.22b or later)

- Typed `Json` codec, `JsonError`, type-app-on-statics, the bare-`Int` tightening → **v0.22b**.
- **Untyped `Json`** value type → later (against the grain).
- **Fetch headers** (`Map[String,String]` on `Response`, 0022) → a separate `karn`-surface
  increment (0023).
- Generic **user**-method chaining → later (built-in method chaining ships **here**).
