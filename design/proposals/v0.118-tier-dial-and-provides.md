# v0.118 — the tier dial (`as unit | integration | system`) and per-seam `provides`; retiring `mocks`, `suite integration`, and `wires` (testing track, slice 6)

- **Scope:** a single-increment **language + test-runtime** change — the seventh testing-track
  slice, and the one that unifies the test *tiers*. It adds two coupled surfaces and removes
  three constructs:
  - an **`as <tier>` clause** (`unit` | `integration` | `system`) on the `case` and `suite`
    header — `unit` is the default and elided — so one test body is *promoted* across tiers
    by changing only its header;
  - a **per-seam `provides`** clause inside a `suite`/`case` — `provides Cap.method(<args>)
    returns <v> | fails` — that overrides *one* collaborator method's provision under test,
    at case or suite scope, including a **sequenced** form for successive calls;
  - and it **retires `mocks`** (the `mocks <Target> = <Impl> { fn … }` re-implementation
    block), the **`suite integration "…" { wires … }`** form (a distinct
    `SourceUnit::Integration`), and the **`wires`** clause — folding integration into `suite …
    as integration` with its participants **inferred** from the `consumes` graph. After this
    the word "mock" is gone from the language. Version → **v0.118**.
- **Realises:** the testing track's *tier is a dial, not a kind* pillar
  (`design/tracks/testing.md`, Pillar IV) — the three tiers borrow the testing pyramid
  verbatim (`unit` collaborators stubbed, `integration` real collaborators within one
  context, `system` contexts wired across the real serialise → JSON → deserialise boundary),
  and a tier is **one body promoted, not a distinct kind of test**. Promotion removes stubs,
  it does not rewrite the body (Principle 5). It also completes the seam triad from DECISION R:
  `consumes` *declares* a seam, `given` *requires* it, `provides` *supplies* it.
- **Relates:** ADR 0144 (one predicate surface — a `provides` arg pattern reuses the
  predicate surface, `_` / literal / `is`, so there is no matcher vocabulary); the capability
  seam and the slice-5 recording proxy (ADR 0152 — observation records at the *same* seam a
  test substitutes at, so `expect Cap.op called …` composes with `provides` and with real
  collaborators unchanged); slice 1b's structural test-ness and build model (ADR 0147 — a
  `suite` is a test-only declaration stripped from the deploy build, so the tier machinery is
  test-build-only for free); the agent-state lifecycle, which this slice pins per tier.
- **ADRs at implementation:** two, numbered at authoring (next free after 0152; do not
  pre-allocate):
  1. **G** — *tier is an `as <tier>` clause on the case/suite header, not a kind of test*
     (folding **K**, participant inference, and **L**, tier-split constraints). Cites ADR
     0144 and DECISION N (the `as` disambiguation, confirmed at settle).
  2. **R + M** — *test-time provider substitution is `provides`, capability-only, at case and
     suite scope* (finalising **V**, the sequenced-return form). Cites ADR 0144.

## The surface

### The tier dial

A `case` runs at one of three tiers, declared with an `as <tier>` clause in its header.
`unit` is the default and is elided; the three names are the testing pyramid:

```bynk
suite money {
  case "never negative"                 { … }  -- as unit, by default
  case "used in a payment" as integration { … }  -- real collaborators, one context, no wire
  case "checkout end to end" as system  { … }  -- contexts wired across the real HTTP edge
}
```

`as` sits on the **suite** header too, setting a default its cases inherit and override
(case wins):

```bynk
suite checkout as integration {          -- every case defaults to integration…
  case "small order authorises"       { … }  -- as integration (inherited)
  case "a unit-level edge"    as unit { … }  -- case overrides the suite default
}
```

Promotion changes only the header — the body is byte-for-byte identical at every tier:

```bynk
case "a small order authorises end to end"                { … }  -- as unit (default)
case "a small order authorises end to end" as integration { … }  -- real Payment, no wire
case "a small order authorises end to end" as system      { … }  -- deployed Workers, real wire
```

At `unit`, a collaborator's own invariants do not run (it is stubbed). At `integration` and
`system` they do — so a green `unit` that fails when promoted means a collaborator's
invariant caught a defect the stub had been hiding, with no new test code.

Tiers are a **`case`-only** affordance. A `property` generates and does not promote
(Principle 5), so a suite-level `as` binds its `case` members only; a `property` under a
tiered suite ignores it (an `as` on a `property` header is `bynk.tier.property_has_tier`).

### Per-seam `provides`

`as <tier>` sets the *default* provision of every seam (unit → doubles, integration/system →
real). A `provides` clause overrides *one* method's provision under test — an explicit call
pattern on the left, a value or `fails` on the right, **never a computed body**:

```bynk
suite pricing {
  provides Rates.lookup("GBP") returns 1.25        -- suite-scoped: applies to every case
  provides Rates.lookup(_)     returns 1.0         -- fallback; first matching clause wins

  case "a fault surfaces as an error" {
    provides Kv.get(_) fails                        -- case-scoped: overrides for this case
    let r <- Prices(Val[AcctId]).quote("GBP")
    expect r is Err(_)
  }
}
```

- The left is `Cap.method(<pattern>, …)` — the capability, the method, and an **argument
  pattern** per parameter. A pattern is the predicate surface: a literal (`"GBP"`, `1000`),
  `_` (any), or an `is` narrowing. Clauses for the same method are tried **top to bottom,
  first match wins**, so put specific before fallback.
- The right is `returns <value>` or `fails` — a *value* or a *fault* (Principle 3: an `Err`
  is an in-band outcome asserted in the case; `fails` injects a capability *fault*). It is
  never a block: a double that needs logic is the signal to promote the tier.
- `provides` is **capability-only** (DECISION M): an agent's realness is the tier's job, not
  a provider's. Overriding a capability the unit does not `consumes` is
  `bynk.provides.not_a_seam` (Principle 4).
- **Precedence:** case `provides` > suite `provides` > the tier default (DECISION M).

Bare **observation needs no `provides`** — the slice-5 recording proxy records calls at the
seam regardless (ADR 0152). You reach for `provides` only when the case depends on a
collaborator's *return*.

### Sequenced `provides` (DECISION V)

A single value cannot express a collaborator whose successive calls differ — an advancing
clock, a `Kv.get` that returns `None` then `Some`, a network that fails twice then succeeds.
The **return-sequence** form supplies one outcome per call, in order:

```bynk
provides Clock.now()  returns each [1000, 2000, 3000]      -- three successive successes
provides Kv.get(_)     returns each [None, Some(row)]        -- None, then Some…
provides Net.fetch(_)  returns each [fails, fails, ok(resp)] -- fails twice, then succeeds
```

- `returns each [<outcome>, …]` — the `each` is what distinguishes "one outcome per call"
  from a single call that returns a *list value* (`returns [a, b]` still returns the two-element
  list once). Each `<outcome>` is a value (a success), the atom `fails` (a fault), or `ok(v)`
  when a fault and a success value must sit in the same sequence.
- **Exhaustion:** the **last outcome repeats** (steady state). `[fails, fails, ok(resp)]` is
  "fails twice, then succeeds forever"; `[1000, 2000, 3000]` holds at `3000` after the third
  call. This describes *the interesting prefix, then hold* and never spuriously fails a case
  that happens to call once more than the author counted (see DECISION V for the rejected
  "error on exhaustion" default).
- A collaborator with genuine internal *state* that must **compute** its next return (a clock
  that keeps advancing by a delta, a paging cursor threaded across calls) exceeds a fixed
  sequence and is the reserved **virtual fixture** — named as a follow-on, **not** shipped
  here (see DECISION V, fixture boundary).

### The agent-state lifecycle, per tier

Fixed across tiers, so promotion is honest: the **unit under test is always a real in-memory
instance, keyed normally, fresh per case**. Only the realness of its *collaborators* and
whether sends cross a serialisation boundary change with the tier. Snapshot and step
invariants are checked at the commit boundary and therefore fire at **every** tier for free —
which is exactly why promoting a `unit` case can surface a collaborator invariant a stub was
hiding.

## What tiers infer, and what constraints attach where (DECISIONS K, L)

- **Participants are inferred, not listed (K).** The compiler already builds the `consumes`
  graph, so `as integration` / `as system` **derive** the real/wired participant set from the
  unit under test's transitive `consumes`. The `suite integration "…" { wires … }` form and
  its `wires` clause are **removed** — one fewer thing to declare, and it cannot
  drift from the real dependency graph. (Open only if a future case shows the participant set
  is genuinely ambiguous; none is known, so slice 6 ships inference.)
- **Constraints split by tier (L).** Today's integration rules — *≥ 2 contexts, no `mocks`* —
  were a property of the one integration construct. With per-case `as` they attach to the tier
  they describe:
  - `system` is the cross-context, wired tier: it stands up ≥ 2 contexts as Workers across the
    real serialisation edge (`bynk.tier.system_needs_wire` replacing
    `bynk.integration.too_few_participants`).
  - `integration` is real collaborators **within one context, no wire** — the missing middle
    between today's unit and integration tests. It carries **no** ≥2-context rule. This
    deliberately *relaxes* today's integration constraints (recorded as a consequence of the
    naming, not an accident).
  - `provides` is legal at **every** tier (it overrides one seam), so the old
    `mock_in_integration` prohibition disappears with `mocks` itself; what a tier controls is
    the *default* provision, which `provides` overrides seam by seam.

## Decisions

**[DECISION G] Tier is an `as <tier>` clause on the case (and suite) header — not a keyword
statement, and not a kind of test.** `unit | integration | system` (the pyramid, for zero-cost
schema transfer); `unit` is the default and elided. The clause sits in the header where
declaration metadata belongs, leaves a clean default, composes at suite and case level (case
wins), and — the load-bearing reason — frames a tier as **one body promoted**. Tiers are a
**`case`-only** affordance; a suite-level `as` binds its `case` members only. Subsumes the
`suite integration "…" { wires … }` form. The tier names are **contextual** in the `as`-clause
position
(like the slice-5 observation words): `unit` / `integration` / `system` stay ordinary
identifiers everywhere else, so nothing that spells `system` today breaks. *Confirmed by
DECISION N — `as` is consumed only in `consumes`-declaration position, and the tier clause is
a distinct case/suite-header production, so there is no ambiguity.* Rejected forms (recorded in
the track under DECISION G): a body-statement keyword (`realism`/`scope`/`tier <level>` — reads
executable but isn't), and a tier-leading case keyword (`integration "…"` — re-instates "three
kinds" in the grammar and reserves the common nouns `unit`/`system`). *Load-bearing ADR.*

**[DECISION M] Per-seam override is `provides`, at case or suite scope; capability-only.** `as
<tier>` sets the *default* provision of every seam; a `provides Cap.method(<args>) returns <v>
| fails` clause overrides one method's provision. Capability-only — an agent's realness is the
tier's job, not a provider's. Precedence: case `provides` > suite `provides` > tier default.
Overriding a non-consumed capability is `bynk.provides.not_a_seam`. *Load-bearing ADR (with R).*

**[DECISION R] Test-time provider substitution is `provides`, not `given` — and `mocks` is
retired.** A test double is an alternative *provider* of a capability, and production already
spells that `provides`. So a stub is a provider scoped to a test — the same seam and mechanism
production uses. This completes the triad: `consumes` declares, `given` requires, `provides`
supplies — `given` reverts to its one production meaning (already the case in the compiler:
`given` = "declare the capabilities a handler requires"). The form is **always an explicit call
pattern**: `provides Cap.method(<pattern>) returns <value> | fails`. Naming the method with its
call pattern keeps the target unambiguous, makes arg-shape matching uniform, and reuses the
predicate surface for the pattern (`_`, literals, `is`) — no matcher words. The right-hand side
is a *value* or `fails`, never a computed body — the rule that stops `provides` becoming the
`mocks` block reborn. This **retires the `mocks` re-implementation block** (a whole alternative
body); together with `Mock[T] → Val[T]` (slice 2, ADR 0148) the word "mock" leaves the language
entirely. *Load-bearing ADR (with M).*

**[DECISION V — finalised] Sequenced `provides` is `returns each [<outcome>, …]`, last-outcome
repeats.** The value-only rule (R) cannot express successive-call differences, which are routine
(advancing clock, `None`-then-`Some`, fail-twice-then-succeed) and where "use the real provider"
fails for fault sequences. **Form:** `returns each [<outcome>, …]`, one outcome consumed per call
in order; each outcome is a value, `fails`, or `ok(v)`. The `each` disambiguates from returning a
list value. **Exhaustion:** the last outcome repeats (steady state) — *rejected* alternatives:
erroring on exhaustion (brittle: an unrelated extra call fails the case on the wrong axis) and
cycling (surprising). **Fixture boundary:** a collaborator that must *compute* its next return
(a delta-advancing clock, a threaded cursor) exceeds a fixed sequence and is the reserved
**virtual fixture** — a named follow-on, not shipped in slice 6. *Recorded in the R+M ADR.*

**[DECISION K — resolved] Tier participants are inferred from the `consumes` graph.** `as
integration` / `as system` derive their real/wired participant set from the unit's transitive
`consumes`; the `suite integration` form and `wires` are removed. *Recommend and ship
inference* — the compiler already has the graph, and an explicit list can only drift from it.
Reopen only against a concrete ambiguous-participant case; none is known.

**[DECISION L — resolved] Constraint regimes split by tier.** The "≥ 2 contexts, no `mocks`"
rules attach to `system` (cross-context, wired); `integration` (real collaborators within one
context, no wire) carries neither. This relaxes today's integration constraints — a deliberate
consequence of the per-case naming, not an accident. `provides` is legal at every tier.

## What changes, against the current code

The `as`-clause and `provides` clause are new; the tier machinery reuses the slice-1b build
strip (ADR 0147) and the slice-5 recording seam (ADR 0152); the removals delete the `suite
integration` form, `wires`, and `mocks`.

1. **Lexer / keywords** — **remove** the `Mocks` token (`lexer.rs:111`) and `Wires` token
   (`lexer.rs:124`), and their registry entries (`keywords.rs:135` mocks, `keywords.rs:193`
   wires). No new hard keyword: the tier names `unit`/`integration`/`system` are contextual in
   the `as`-clause (as `integration` is already a contextual `Ident`, detected at
   `parser/declarations.rs:54-78`/`:830`); `provides` is the existing keyword (`lexer.rs:157`,
   "Provide an implementation of a capability"), gaining the test-scope per-method form,
   distinguished from the production form by `.method(` after the capability name vs `=
   Provider`; `as` is the existing `As` token (`lexer.rs:106`), reused in header position
   (DECISION N — its sole current consumption is `parse_consumes_decl` at
   `parser/declarations.rs:462`). `given` is unchanged (`keywords.rs:109`) — already
   production-only "require", so the triad needs no change to it.
2. **AST / parser** — add an optional `as <tier>` to the `case` and `suite` headers
   (`parse_test_case` at `declarations.rs:1084`; `parse_test_brace`/`parse_test_fragment` at
   `declarations.rs:555`/`:682` — the current headers are string-literal-only, so this is a
   new attachment point), a `TestTier` enum (`Unit`/`Integration`/`System`) on `Case`
   (`ast.rs:301`) and the suite, and a `ProvidesClause` list (method path + arg patterns +
   `returns`/`returns each`/`fails` RHS) at suite and case scope. **Remove** `parse_integration`
   (`declarations.rs:824`), `parse_wires_clause` (`declarations.rs:989`), the
   `SourceUnit::Integration` variant + `IntegrationDecl` (`ast.rs:242`/`:352`), and the
   `mocks`-block parse `parse_mock_decl`/`parse_mock_op` (`declarations.rs:1011`/`:1055`) with
   `MockDecl`/`MockOp` (`ast.rs:276`/`:288`). Reuse the predicate/`is` parser for arg patterns
   and the value-expression parser for the RHS.
3. **Checker** — resolve a `provides` target against the unit's consumed capabilities (reject
   non-seam `bynk.provides.not_a_seam`, unknown op `bynk.provides.unknown_op`, an RHS whose
   type disagrees with the op result, a malformed sequence `bynk.provides.bad_sequence`); check
   tier legality (`bynk.tier.property_has_tier`, `bynk.tier.system_needs_wire`); infer
   participants from the `consumes` graph for `integration`/`system`. **Remove** the
   `bynk.integration.*` family (`diagnostics.rs:510-537`: `mock_in_integration`,
   `too_few_participants`, `unknown_participant`, `unwired_dependency`, `duplicate_participant`,
   `duplicate_suite`) and the `bynk.mock.*` family (`tests_emit.rs`: `duplicate_target`,
   `unknown_target`, `signature_mismatch`, `in_commons_test`), superseded by the tier /
   `provides` diagnostics.
4. **Emitter** (`bynk-emit/src/project/tests_emit.rs`) — the mock-resolution phase
   (`tests_emit.rs:135-239`) becomes **provides-resolution**: resolve each seam's provision from
   the tier default + `provides` overrides (case > suite > default), lowering a `provides`
   clause to a stub object behind the recording proxy (ADR 0152): a `returns <v>` to a constant
   return, `fails` to a thrown/`Err` fault, `returns each [...]` to a per-call cursor with
   last-repeat exhaustion. For `integration`/`system`, stand up the inferred real collaborators
   (system across the serialisation edge). Fold the integration emit into `emit_test_module`
   (`tests_emit.rs:270`). Test-build-only via the ADR-0147 strip.
5. **Diagnostics** — a `bynk.tier.*` family and the `bynk.provides.*` family
   (`not_a_seam`, `unknown_op`, `rhs_type`, `bad_sequence`); retire `bynk.integration.*` and
   `bynk.mock.*`.
6. **tree-sitter** (`tree-sitter-bynk/`) — add the `as <tier>` clause to `suite_decl`/`case`,
   a `provides_clause` (method call pattern + `returns`/`returns each`/`fails`), the contextual
   tier words and `each` to `highlights.scm`; remove the `integration_decl`/`wires` rules; new
   corpus; re-bless `grammar.json` and the embeddable-rule count. **vscode-bynk** — the tier
   words and `provides` RHS; drop `mocks`/`wires`/`integration` unit highlighting.

## Migration (clean-slate churn, with codemods)

- **`mocks` block → `provides` clauses.** A re-implementation block

      mocks Logger = SilentLogger { fn log(msg: String) -> Effect[()] { () } }

  becomes one `provides Cap.method(<pattern>) returns <value>` per method — here `provides
  Logger.log(_) returns ()`. A `mocks` method whose body has *logic* (branches, reads an
  argument to compute the result) has no value form — the codemod flags it for the author to
  either promote the case to `as integration` (use the real collaborator) or express it as a
  `returns each` sequence. After migration the `Mocks` keyword is dropped; "mock" is gone from
  the language.
- **`suite integration "n" { wires C1, C2; case … }` → `suite n as system { case … }`.**
  Participants are inferred from `consumes`, so `wires` is dropped. A single-context test that
  wired only its own context becomes `as integration`; a genuine cross-context one becomes `as
  system`. The parse-time `mock_in_integration` rejection and the stale note it carries (which
  still references the retired `test <context> { mocks … }`, `declarations.rs:922-939`) are
  removed with the form.
- **Book & examples** — `guides/testing/integration.md` is absorbed into the tier-dial guide;
  every `mocks`/`wires`/`suite integration` snippet across the book and
  `design/tracks/testing-example/` is rewritten to `as <tier>` + `provides`.
