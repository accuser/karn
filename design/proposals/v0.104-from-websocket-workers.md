# v0.104 — `from WebSocket` on Workers: the Durable Object hibernatable upgrade (real-time track, slice 3b)

- **Scope:** a single increment — the **Cloudflare Workers** wire path for the
  `from WebSocket` protocol: the edge-authenticated upgrade, the socket accepted
  **into the addressed Durable Object** (the agent), the hibernatable lifecycle,
  inbound frames dispatched as typed agent messages, and the `WorkersConnection`
  runtime. Removes the slice-3a platform-lock
  (`bynk.target.websocket_workers_unsupported`). **Emitter + runtime, Workers
  target.** No language-surface change — the same `from WebSocket` source that
  runs on bundle (3a) now deploys to Workers.
- **Posture: the track's safety boundary, runtime half.** This is where the
  edge-auth-before-accept (3a's compile-time guarantee) becomes a **running
  fail-closed gate**, and where the **`Sec-WebSocket-Protocol` credential read**
  (D-B's runtime half) lives — the single most security-sensitive code in the
  track. Carries the threat model (§5) and runs **`/security-review` +
  `/code-review`**, per [ADR 0076](../decisions/0076-feature-track-posture.md).
- **Realises:** design notes §7 ("the upgrade happens in the Worker; the server
  socket is accepted into the addressed Durable Object … survives hibernation") and
  `bynk-type-system.md` §2.9.6 (hibernation transparency). Completes the **Workers**
  half of [ADR 0131](../decisions/0131-from-websocket-protocol-bundle.md) and the
  retained proposal `v0.103-from-websocket-protocol.md`. Settles **Q7**
  (hibernation re-acquisition) and the **runtime half of Q8** (the auth carrier).
- **Builds on:** ADR 0131 (the protocol surface + checker + bundle vertical — the
  `on open` rules, `by`-mandatory, Signature-rejected, the `Connection` linearity
  are all in place); the Workers agent→Durable-Object machinery (`makeWorkersAgent`,
  the DO `fetch` dispatcher, `wrangler.toml` DO bindings + migrations); the Bearer
  seam (`bearer_seam_for`, `verifyBearerJwtHs256`) the HTTP edge already uses.
- **Authorises its own ADR:** the Workers upgrade topology, the subprotocol auth
  carrier, the hibernation mapping, and the inbound-frame dispatch. **Number at
  PR-cut** (provisionally 0132).

## 1. The topology — authenticate at the edge, accept into the agent's DO

The design note's sentence hides the whole problem: *"the upgrade happens in the
Worker … the server socket is accepted into the addressed Durable Object."* A
live `WebSocket` cannot be passed over the DO RPC, and **hibernation requires the
socket to be accepted by the DO that hosts it** (`state.acceptWebSocket`). So the
upgrade *request* is what moves, not the socket:

```
client ──upgrade──▶ Worker.fetch
                      │ 1. match the `from WebSocket` service
                      │ 2. AUTHENTICATE the actor (read the Bearer token from the
                      │    Sec-WebSocket-Protocol subprotocol; verify fail-closed)
                      │ 3. resolve the target DO from the on-open transfer
                      │    (`Room(roomId)` → the Room DO, keyed by a request param)
                      ▼
                 stub.fetch(request, {verified identity})   ── forward, only on auth success
                      │
              Room DO.fetch
                      │ 4. new WebSocketPair(); state.acceptWebSocket(server, [connId])
                      │ 5. run the `on open` body in the DO: WorkersConnection(server),
                      │    `connection.send(welcome)`, then the agent-local `join`
                      ▼
                 return 101 (client socket)  ──▶ client
                      ⋮
client ──frame──▶ Room DO.webSocketMessage(ws, msg)
                      │ decode msg against `in:`; dispatch to the agent handler
```

The **`by` actor verifies in the Worker, before the request is forwarded** — no
unauthenticated request reaches the DO, no socket is accepted before auth. That is
the safety boundary made runtime-real.

## 2. Scope and non-goals

**In scope.**

- The **Worker-edge upgrade**: detect `Upgrade: websocket`, match the WS service,
  run the `by` auth seam reading the token from **`Sec-WebSocket-Protocol`**
  (D-B), and on success forward the request (carrying the verified identity) to
  the addressed DO; fail-closed `401`/`426` otherwise.
- **The Durable Object as the connection host**: the agent the `on open` transfers
  to grows the hibernatable handlers — its `fetch` does the `WebSocketPair` +
  `acceptWebSocket`, and `webSocketMessage`/`webSocketClose`/`webSocketError`
  decode an inbound frame against `in:` and dispatch to the agent's handlers
  (and dispose the held connection on close).
- **`WorkersConnection`** — the runtime `Connection<F>` for Workers: `send(frame)`
  → `ws.send(<out-codec>(frame))`, `close()` → `ws.close()`.
- **Hibernation re-acquisition (Q7)** — `acceptWebSocket(server, [connId])` +
  `serializeAttachment`; on wake `getWebSockets(connId)` re-presents the socket and
  the stored `Connection` re-wraps it, **entirely in the runtime binding**, behind
  the `Connection<F>` interface (the language sees a stored value, §2.9.6).
- **Removing the platform-lock** (`validate.rs`), the `wrangler.toml` DO-migration
  for a connection-hosting agent, and the Worker DO re-export.
- The **§20 chat-room emitting + `tsc --strict`-clean on Workers**, and a
  `workerd`/Miniflare behaviour test if the harness supports it (else a
  shape-snapshot fixture + tsc).

**Non-goals (and why).**

- **The broadcast-to-all-connections runtime** (`connections.values.parTraverse(c
  => c.send(…))`) — the held-aware iteration borrow surface, deferred since slice 2;
  lands with the **slice-4 closure**. 3b hosts and dispatches a single connection's
  lifecycle; the chat-room's fan-out is slice 4.
- **Multiple / conditional connection-transfer targets in one `on open`.** v1
  requires the `on open` to transfer the connection to **exactly one** agent whose
  key is derivable from the request (D2); richer routing earns its own slice.
- **Non-Cloudflare WebSocket platforms** — `Connection` is a Cloudflare-DO-hosted
  resource here; other platforms are a Tier-3 binding concern (§18), out of scope.
- **A streaming `Ai` consumer / `Queue`-out** — track non-goals throughout.

## 3. Decisions

**[DECISION A — the topology] Edge-auth in the Worker, accept-into-the-DO; the
held-connection transfer compiles to a DO upgrade-forward.** The `on open` body
runs **in the Durable Object** that hosts the connection (where, on Workers, the
agent is local). The Worker authenticates and forwards the upgrade request to that
DO; the DO accepts the socket and runs the body. The **agent-transfer of the
on-open connection** (`Room(roomId).join(…, connection)`) is the operation the
emitter recognises and lowers to *"this is the hosting DO; accept here."*
*Recommended;* it is the only shape that satisfies both Cloudflare hibernation
(socket in the DO) and the language semantics (the handler picks the DO).

**[DECISION B — the on-open shape constraint] At v1 the `on open` transfers the
connection to exactly one agent, keyed by a request-derivable value.** The
emitter resolves the target DO statically from that single transfer; `connection.send`
and other ops run in the DO after accept. A handler with zero or multiple
connection-transfers, or a non-derivable key, is a compile error
(`bynk.ws.open_transfer_shape`). *Recommended;* it keeps the edge→DO routing
static and is exactly the §20 chat-room shape. Conditional/multi-target routing is
a named follow-on.

**[DECISION C — Q8 runtime, the security crux] The Bearer token is read from the
first `Sec-WebSocket-Protocol` element, verified at the edge before forwarding, and
the accepted subprotocol is echoed on the `101`.** A browser sets it via `new
WebSocket(url, [token])`. The Worker: `request.headers.get("Sec-WebSocket-Protocol")`
→ first comma-separated element → `verifyBearerJwtHs256(token, secret)` (the same
verifier HTTP uses) → on `Err`, **`401`, do not forward**. On success the `101`
response sets `Sec-WebSocket-Protocol: <token-subprotocol-name>` so the handshake
completes. *This is the call `/security-review` scrutinises hardest;* it reuses the
audited JWT verifier, changes only the *carrier*, and stays fail-closed.

**[DECISION D — Q7, hibernation transparency] Re-association lives in the runtime
binding, keyed by a connection id via `serializeAttachment`.** `acceptWebSocket(server,
[connId])` tags the socket; `ws.serializeAttachment({ connId })` persists it across
hibernation. A stored `Connection` is the `connId`; on wake the binding resolves it
via `state.getWebSockets(connId)` (or `deserializeAttachment` on dispatch) and
re-wraps the re-presented socket in a `WorkersConnection`. The language keeps
"stored value in, stored value out" (§2.9.6) — the indirection never surfaces.
*Recommended;* if the id→socket resolution cannot be fully hidden, that is the one
place "transparent" leaks, flagged for the platform-mapping review.

**[DECISION E — inbound frames] `webSocketMessage(ws, msg)` decodes `msg` against
`in:` (the boundary-decode discipline) and dispatches to the matching agent
handler; an undecodable frame is rejected, never dispatched.** `webSocketClose`
disposes the held connection (the linearity discipline's runtime counterpart — the
stored connection is removed-and-closed). *Recommended;* it is the §7 "frames are
typed messages" model, with the decode as the trust boundary on client bytes.

## 4. Internal architecture

- **`emit_websocket_upgrade`** (`workers.rs`, today a `503` stub) becomes the edge
  route: the subprotocol auth read (C), the target-DO resolution (A/B), and
  `stub.fetch(request, …)` with the verified identity threaded in a header the DO
  trusts (the same Internal-channel trust the cross-context `X-Bynk-Caller` seam
  uses).
- **The connection-hosting agent's DO class** (`emit.rs`) grows
  `fetch` (the `WebSocketPair`+`acceptWebSocket` on the upgrade path) and
  `webSocketMessage`/`webSocketClose`/`webSocketError`, alongside its existing
  `/_bynk/agent/<method>` RPC dispatch.
- **`WorkersConnection`** is added to `connection.ts` (beside `TestConnection`),
  serialising frames through the `out:` type's codec.
- **`wrangler.toml`**: the connection-hosting agent's DO migration already exists
  (agents are DOs); no new binding kind. The platform-lock check is deleted.

## 5. Security & threat model (the `/security-review` brief)

- **Unauthenticated socket into an agent.** *Defence:* A — the Worker verifies the
  actor and **only forwards on success**; no socket is accepted before auth.
  *Review focus:* there is no path from `fetch` to `stub.fetch`/`acceptWebSocket`
  that skips the seam, and a forwarded request cannot itself bypass (the DO trusts
  only the Internal channel + the verified-identity header, never a client header).
- **Auth-carrier confusion (C).** A token in the wrong place, a spoofable identity
  header, or a malformed subprotocol. *Defence:* read only the first
  `Sec-WebSocket-Protocol` element, verify with the audited JWT verifier, echo the
  subprotocol; the DO accepts the **verified identity only from the trusted edge
  header**, never from the client. *Review focus:* the identity header cannot be set
  by the client (stripped/overwritten at the edge); malformed/oversized subprotocol
  → `401`, not a crash.
- **Trusting client frame bytes (E).** *Defence:* `webSocketMessage` decodes against
  `in:` before dispatch; undecodable → drop. *Review focus:* the decode is total.
- **Hibernation re-association integrity (D).** A wrong `connId`→socket mapping
  could cross-wire two clients' sockets. *Review focus:* `connId` is unguessable and
  per-connection; `getWebSockets(connId)` returns only that socket.
- **Connection-leak on abnormal close.** `webSocketClose`/`webSocketError` dispose
  the stored connection (the linearity runtime counterpart), and a faulted handler
  rolls back agent state (ADR 0109).

## 7. Risks & mitigations

- **The upgrade-forward topology is intricate** (edge → DO, identity handoff,
  socket-in-DO). *Mitigation:* D2 constrains the `on open` to the one statically-
  routable shape; the §20 chat-room is the proof; the auth seam reuses the audited
  HTTP path unchanged but for the carrier.
- **The security crux (C) is the track's sharpest edge.** *Mitigation:* `/security-
  review`-gated; the threat model (§5) is explicit; only the carrier changes, the
  verifier is reused.
- **Hibernation transparency could leak (D/Q7).** *Mitigation:* all re-association
  in the runtime binding behind `Connection<F>`; a hibernate/wake behaviour test (or
  a documented `workerd` limitation) proves it.
- **`workerd` test harness availability.** A real hibernation test may not be CI-
  portable. *Mitigation:* fall back to a tsc-verified shape snapshot + a documented
  manual `wrangler dev` check; never claim a runtime proof that did not run.
